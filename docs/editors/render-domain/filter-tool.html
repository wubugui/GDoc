滤镜工具

同一座码头，日头下和雨夜里该是两种气色。滤镜工具用滑条可视化调色彩矩阵滤镜，实时对比原图和调色后的效果，保存预制后游戏直接加载——它和主编辑器滤镜面板看的是同一批数据，两边改一处、另一边刷新即见，谁用起来顺手就用谁。

这是什么（30 秒看懂）​

滤镜工具本质是一台"调色台"：拖几条滑条，就能让一张场景截图整体偏冷、偏暖、变暗、褪色，效果和游戏里加载滤镜后看到的完全一致——因为它导出的格式就是游戏滤镜加载器直接认的那种，不需要你手改任何数据文件。

它解决的问题是：与其在脑子里凭空想象"雨夜偏冷一点该是什么感觉"，不如拿真实的场景截图当画布，边拖滑条边看，调对了就存成一个有名字的预制（比如 wujin_rain_night），以后场景、过场想用这个色调，直接挂这个预制 id 就行，不用每次重调。

它只管全屏统一色调，不做局部光照（那是光照体积烘焙的事），也不改场景的遮挡碰撞（那是场景深度重建的事）。

入门：手把手做第一次​

  1. 命令行跑 ./dev.sh filter-tool 打开工具窗口；也可以从 Web 控制台点"滤镜工具"，或主编辑器工具 → 外部工具（新进程） → Filter Tool 打开。
  2. 点加载图片，选一张能代表目标氛围的场景截图或背景图——比如要调雨夜码头的色调，就加载码头场景的截图。
  3. 先从预制里选一个相近的起点：内置有还原、夜晚、黄昏、阴天、复古五种，也能选自己之前存过的自定义预制。选中后点击应用，作为微调的起始状态，比从零拖滑条快得多。
  4. 拖四条滑条微调：亮度（0.2–2.0，越小越暗）、对比度（0.2–2.0）、饱和度（0–2.0，0 即黑白）、色相（-180 到 180，整体色相偏移）。中间预览区实时显示调整后的效果，心里钉住目标场景的氛围——城隍庙夜祭偏暖黄，义庄阴风偏青灰。
  5. 满意后在输入框填个名字，点添加当前为预制（对应"保存为新预制"这一步），预制会自动存进工具的自定义预制列表；如果是要正式给游戏用的滤镜，再点保存滤镜，输入滤镜 id，写入工程的滤镜数据目录——这个 id 一旦保存就相当于文件名，不建议后续改名。
  6. 回主编辑器滤镜面板或场景面板，挂上这个滤镜 id。
  7. F5 运行预览进对应场景，确认氛围符合预期；如果场景同时挂了光照体积，留意两者叠加后别过曝或死黑。

进阶：把每一项都讲透​

四个调色参数

  • 亮度（Brightness）：整体明暗，范围 0.2–2.0，默认 1.0（不变）。数值越小画面越暗，适合夜景、阴暗室内；数值越大适合正午强光场景。
  • 对比度（Contrast）：明暗反差，范围 0.2–2.0。调高让明暗更分明、画面更"硬朗"；调低让画面更"糊"，常用来做回忆、朦胧氛围。
  • 饱和度（Saturation）：色彩鲜艳程度，范围 0–2.0，0 即完全去色变黑白。义庄、阴风场景常会把饱和度往下压一些，让色彩显得压抑。
  • 色相（Hue）：整体颜色偏移，范围 -180 到 180 度，相当于把色环整体旋转。偏冷（青蓝）还是偏暖（橙黄）主要靠这个参数配合亮度/饱和度一起调。

预制系统

  • 内置预制：还原（无任何调整的原图）、夜晚（整体压暗压蓝）、黄昏（偏暖偏低对比）、阴天（整体降饱和偏灰）、复古（偏暖偏黄有陈旧感）——五种是最常见氛围的现成起点，新建预制时优先从这五个里选一个最接近的开始微调，而不是从"还原"直接拖到位。
  • 添加当前为预制：把当前调好的四个参数组合存成一条自定义预制，存在工具自己的本地预制列表里（不等同于正式的滤镜 id），方便你在同一次会话里反复套用、对比。
  • 自定义预制备份：自定义预制列表是工具本地存的，换机器或重装环境前，记得把已经用得上的预制正式保存滤镜导出到工程里，否则本地列表换机器就没了。

保存与 id

  • 保存滤镜：把当前调色结果写成一份正式的滤镜数据，存进工程约定的滤镜目录，id 就是你输入的名字。
  • 滤镜 id 即文件名：保存后这个 id 相当于文件名，改名要格外小心——所有引用它的场景、过场都要跟着同步改，否则会出现"滤镜丢了"的报错或效果对不上。
  • 和滤镜面板同源：滤镜工具存下的预制、滤镜工具打开时能读到的自定义列表，都和主编辑器滤镜面板读写的是同一份工程数据。在滤镜面板上可以直接打开滤镜工具做精细调色，或者用"从磁盘重载"同步另一边保存的最新结果。

和别的工具/面板配合

  • 挂哪个场景、哪个过场用这个滤镜 id，是场景面板和过场的活，滤镜工具本身不负责"引用关系"，只负责"调出一个滤镜数据"。
  • 和光照体积烘焙是互补关系：滤镜管全屏统一色调，光照体积管局部动态光源。两者会叠加显示，调色时如果已经知道这个场景会挂体积光，建议把滤镜调得稍微保守一点，留出体积光提亮的空间，避免叠加后过曝。
  • 和场景深度重建完全无关：滤镜不改遮挡关系，穿模问题回场景深度工具解决，不要指望调色能掩盖穿模。

效率窍门

  • 全工程如果想要统一的"晨雾"基调，做一个预制多个场景复用，比每个场景各调一份更省心，后续要整体调整氛围时也只需要改一份。
  • 只是想换场景挂哪个已有滤镜，不需要开工具，直接在主编辑器滤镜面板或场景面板里选就够了。
  • 调色矩阵调"疯"了（画面完全失真）不用一点点往回拖，直接重新应用一个相近的内置/自定义预制作为新起点。

什么时候用它 / 和别的工具配合​

情况建议
新场景需要独特色调新建一个预制，调好后再挂场景
全工程想统一晨雾色/统一夜色做一个预制，多场景复用同一个滤镜 id
只是想改场景挂哪个滤镜主编辑器滤镜/场景面板即可，不必开工具
调色矩阵调"崩"了重新应用一个相近预制再微调，而不是逐格拖回去
场景同时要局部动态光滤镜先调保守一些，留空间给光照体积叠加

边界与当心

  • 和体积光、后期叠得太狠：画面容易死黑或死白，调色时留有余地。
  • 预制 id 改名：所有引用处（场景、过场）要同步改，不然会出现引用不到的问题。
  • 自定义预制只在本地：换机器前记得把用得上的正式保存到工程，别只留在工具本地列表里。
  • 滤镜不等于深度：滤镜工具不管遮挡和碰撞，穿模问题要回场景深度重建。

常见问题​

Q：为什么保存的滤镜进游戏后颜色和工具里预览的不一样？ A：先确认保存滤镜这一步真的执行了（不是只点了"添加当前为预制"）；预览用的是工具内加载的截图，实际场景光照、其它叠加效果可能让最终观感略有差异，属于正常范围，微调即可。

Q：内置预制和自定义预制有什么区别？ A：内置的五种（还原/夜晚/黄昏/阴天/复古）是工具自带、不能删改的起点模板；自定义预制是你自己调好存下来的，存在工具本地列表里，需要正式保存滤镜才会进工程。

Q：滤镜 id 能不能改名？ A：技术上能重新保存成新 id，但旧 id 不会自动消失，且所有引用旧 id 的场景/过场都得手动改成新 id，容易漏改，不建议频繁改名。

Q：饱和度调到 0 是什么效果？ A：完全去色，画面变成黑白，可以用来做闪回、记忆片段这类特殊叙事氛围。

Q：色相和亮度/对比度/饱和度要怎么配合调？ A：一般先用亮度和饱和度定"暗不暗、艳不艳"的大方向，再用色相微调冷暖倾向，最后用对比度调画面的"硬朗"或"柔和"感,顺序不是死规定,但这样调起来更顺手。
