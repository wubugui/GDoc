# 视差场景编辑器

过场里图层缓缓拉开、镜头似在雾中穿行——**视差场景编辑器**在浏览器里编 **图层、关键帧、运动轨迹**，渲染方式与游戏过场一致，所见即所得。结果写入工程视差场景数据，供过场步骤 **呈现视差场景** 调用。

---

## 干什么

- 管理过场用的 **视差场景** 条目（列表选场景 id）。
- **图层**堆叠：远景、中景、前景，各层可独立图。
- **关键帧**与 **轨迹**：控制每层位移、缩放、透明度随时间变化。
- **预览播放**整条过场时间轴。
- **保存**到 `parallax_scenes` 类数据（游戏直接读）。

---

## 怎么开

**方式一：命令**

```bash
./dev.sh parallax-editor
```

浏览器打开（常见端口 **5205**）。若同端口已有实例，会**复用**不重复起服。

**方式二：指定场景**

```bash
./dev.sh parallax-editor
```

启动后选场景，或按终端说明带场景 id 参数直接打开某条。

**方式三：Web 控制台**

点 **视差编辑器**。

**方式四：主编辑器**

**工具 → 外部工具** → **视差编辑器**；过场编排里 **呈现视差场景** 步也可能有快捷打开按钮。

---

## 一步步怎么用

1. 启动视差编辑器，列表选场景或新建 id（如 `shenxianding_02_demo`）。
2. **添加图层**，每层绑定已 [入库](../asset-domain/asset-ingest) 的图。
3. 设每层 **视差系数**（远景慢、前景快）。
4. 时间轴上打 **关键帧**：起点/终点位置、可选缩放与透明度。
5. 拖播放头 **预览**整条，修跳变与穿帮。
6. **保存**。
7. 主编辑器 **过场** 面板某步选 **呈现视差场景** 并指向此 id，F5 或过场预览验证。

---

## 何时用

| 情况 | 建议 |
|---|---|
| 新做过场卷轴镜头 | 在本工具编轨迹 |
| 过场里图层错位、节奏不对 | 回时间轴改关键帧 |
| 静态对话过场 | 不必视差，用过场别的步类型 |
| 只换一张层图 | 改图层资源路径，关键帧可保留 |

---

## 当心什么

| 当心 | 说明 |
|---|---|
| 图层分辨率差太大 | 放大糊、缩小锯齿 |
| 总时长与过场脚本不一致 | 过场切下一步时视差还没播完 |
| id 与过场步填的不一致 | 游戏里黑屏或播错场景 |
| 未保存关浏览器 | 同端口复用有时仍见旧缓存，保存后刷新 |

## 雾津例子

1. 新建 `fog_dock_intro`：三层——远雾、中码头、近缆桩。
2. 0–5 秒远景慢移，前景缆桩略快，营造进站感。
3. 保存后过场 `prologue_enter_dock` 插入呈现视差步，指向 `fog_dock_intro`。
4. 新章开场播放，与 [滤镜](./filter-tool) 晨雾色调一致。
5. 教程 [视差过场](../../tutorials/parallax) 可对照练习。
