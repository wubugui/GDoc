# 角色照明实验室

雾津的角色不能再当「贴在背景上的自发光贴纸」。茶馆烛火侧的暖边、义庄屋檐下的暗部、河滩雾气里的背光，都要求角色把立绘当**反照率**，从伪世界里采一份辐照度再着色。**角色照明实验室**是这套管线的离线工作台：重建伪几何 → 烘焙探针 / 体素 → 导出正典深度与游戏照明载荷。

原理与账本（推导 → 运行时同式）见开发文档 **[伪世界角色照明](../../dev/character-lighting)**。本页只讲**怎么开、怎么烘、怎么导出、怎么验收**。

---

## 这是什么（30 秒看懂）

把实验室想成雾津的暗房：

1. 一张场景背景进来；
2. 估深度、标定伪世界、补占据与辐射体素；
3. 在可达高度带里布探针，半球 gather 成 L1 / L2 / BIN 缓存；
4. 把深度与照明载荷导出给游戏；
5. 游戏里角色逐像素走与查看器**同一套**着色式（不是另做一套 tint）。

| 你要做的事 | 在本工具 |
|---|---|
| 伪世界重建 / 重烘 | 应用参数重烘当前 / 重烘全部 / 新场景入库 |
| 对照 RT 与探针 | Tab 或 1–4 切换 RT / L1 / L2 / BIN |
| 改几何（抬高、压平、碰撞） | 圆刷 / 多边形选区；改完必须重烘 |
| 写出游戏可用深度 | **⇪ 导出场景深度** |
| 写出游戏可用照明 | **⇪ 导出照明** |

无烘焙或哈希过期的场景，游戏自动回落旧光环境曲线——玩家无感切换。

## 怎么打开

```bash
./dev.sh char-lighting
# 或固定端口、不自动开浏览器：
./dev.sh char-lighting -- --no-open --port 5311
```

Web 控制台亦可点 **角色照明实验室**。

CLI 脚本化入库（备用）：

```bash
./dev.sh char-lighting -- --build <背景图.png> --name <场景名>
```

日常烘焙优先用页内按钮（异步队列 + 实时日志，可排队可离开页面）。

## 入门：第一次把城隍庙 / temple 打通

### 1. 打开并选场景

打开实验室，从下拉选已有场景，或 **＋ 新场景入库…** 选一张背景图。入库会按当前重建参数跑完整管线：深度 → 标定 → mesh → 体素 →（可选）语义门控 → 探针 → 碰撞。

### 2. 看角色站进光里

方向键 / WASD 移动角色；点画面瞬移（开碰撞则吸附最近可走点）。滚轮缩放、右键平移 2D 视图。

用 **Tab** 或数字键在四种模式间切换，确认烛火边 / 暗角观感接近：

| 键 | 模式 | 用途 |
|---|---|---|
| 1 | RT | 参考路径（慢，真值） |
| 2 | L1 | 低阶球谐缓存 |
| 3 | L2 | 生产默认（≈无损于 Lambert） |
| 4 | BIN | 无球谐截断的方向 bin |

### 3. 改参数后重烘

重建区可调俯角、ppu、起伏增益、场景 EV、HDR 最大 EV、是否烘焙折叠、语义门控等。改完点 **⟳ 应用参数重烘当前场景**。几何笔刷改过深度 / 碰撞后会出现「几何已改需重烘」——此时导出按钮会拒绝，直到重烘完成。

### 4. 导出两份给游戏

顺序建议：

1. **⇪ 导出场景深度** —— 正典深度 + 深度配置写回场景（含碰撞网格等；手调容差类参数保值照抄）
2. **⇪ 导出照明** —— 写入运行时 `lighting/` 载荷，并把当前面板着色参数写入配置块

### 5. 进游戏验收

起游戏或主编辑器 **F5**。有载荷的场景：角色 / NPC 走烘焙着色。按 **F2** 打开「角色照明（烘焙）」：模式、NEE、折叠、miss、β、隆起等与导出配置一致；这里改只是临时测试，重载场景会回到导出值。

雾津例子：义庄门口交差位——侧上方灯笼暖边、背光不吃直接烛火但仍有墙地环境、从河滩走进门口时亮度连续变化。

## 进阶：面板与旋钮怎么用

### 四个模式与 miss 闭包

探针烘焙把缓存拆成命中贡献 / miss 环境贡献 / 覆盖率。运行时：

- **miss 环境强度**：`E_hit + s·E_amb`（`s=0` 严格黑）
- **miss 不计入**：按覆盖率归一
- **折叠开着时** miss≈空，上述旋钮会「没反应」——要对比闭包差异，重建区关掉「烘焙折叠」再烘一次

### NEE 直接光

发光增量只走显式光源采样；普通半球命中只吃基础辐射。开关 NEE 时期望均值几乎不变是分账正确；观感集中在贴近光源的部位（脚边烛火照腿）。要放大发光核：重建区把 HDR 最大 EV 拉高后重烘。「光源」勾选可在 2D/3D 检视提取结果。

### 几何编辑

- 圆刷：抬高 / 压低 / 抹平到地面 / 平滑 / 碰撞可走 / 阻挡 / 擦除
- 多边形选区：单击加点、双击闭合、Esc 取消
- 编辑持久保存在场景输出目录，重烘自动应用

### 3D 检视

轨道相机查看可见壳 / 补全层 / 地面延拓；探针球颜色≈其低频亮度；射线黄=直接命中、橙=折叠、蓝=miss。网格着色从世界坐标反投影采样原图——不是顶点色糊弄。

### 预览亮度滑杆

纯显示增益（±EV），不污染物理 `β` 与场景曝光。暗场景看不清先拉它。

### 与旧场景深度工具的分工

| 目标 | 推荐 |
|---|---|
| 角色照明、探针、导出照明载荷 | **本实验室（主路径）** |
| 导出正典场景深度（含几何编辑） | **本实验室** |
| 只查遮挡 / 旧流程对照 | [场景深度重建](./scene-depth-editor)（逐步 deprecated） |

一句话：照明与深度导出以实验室为准；旧深度工具不再是照明的平行数据源。

## 危险区与边界

- **几何 / 曝光 / 折叠 / 增益改了必须重烘**，否则导出被拒或游戏哈希门禁用烘焙。
- **导出照明会清掉旧 v1 载荷**；确认整队已切 v2。
- **立绘当 albedo**：暗场景角色就是暗；不要用把角色「提亮成贴纸」的方式骗融入——该调的是曝光、增益、β 与站位。
- **白天室外方向太阳**仍是游戏侧 F2 独立项，不是探针烘焙的一部分。
- **薄结构**在体素分辨率下会变粗；细栏杆别指望厘米级遮挡。
- **实验室接触阴影不进游戏**；游戏阴影 / AO / 深度遮挡走原系统，勿指望这里「一次调完所有影」。

## 常见问题

**游戏里角色观感和实验室差很多？**  
先确认已「导出照明」且背景哈希未过期；再确认游戏开着烘焙着色（F2）。两边必须同模式（默认 L2）、同 NEE / 折叠设定。仍差则开 RT 对照，查是 reference 错还是缓存错——禁止先拧缓存糊弄过去。

**背对蜡烛角色全黑？**  
不该。墙地基础辐射应仍在。查主法线、折叠、miss 强度，以及是否误开了「严格黑」类策略。

**角色上半身穿进前景柱子？**  
遮挡代理问题：确认导出的场景深度与碰撞；着色直立与遮挡 billboard 已在管线拆开——若仍穿，回实验室几何编辑抬 / 断前景壳。

**改了 bulge / β，重进游戏又回去了？**  
非 bake 参数在「导出照明」时写入场景配置；只在 F2 拧不等于写回。要固化：在实验室调好 → 再导出照明。

**无烘焙场景角色发灰 / 发飘？**  
正常回落旧光环境曲线。该场景尚未导出照明，或哈希失配被禁用。
