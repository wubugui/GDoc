角色照明实验室

雾津的角色不能像纸片一样平贴在背景上。茶馆门口的灯、义庄屋檐下的阴影、河滩水雾里的背光，都会影响角色看起来是不是站进了同一个世界。角色照明实验室就是给你做这层「角色如何被场景照亮」的工作台：先把场景整理成可用的伪世界，再调角色光照、探针和导出数据。

这是什么（30 秒看懂）​

角色照明实验室处理的是角色与场景光照如何合在一起。你可以把它想成一间小暗房：背景图进来后，不只看它长什么样，还要估它的空间前后、哪里该亮、哪里该暗、角色站进去时该受哪种光。

它主要做四件事：

用途你用它做什么
伪世界重建把平面场景整理成可供角色采样的空间感与光照参照
角色光照调角色在当前场景里的受光、阴影、融入程度
场景入库把整理好的场景资料纳入工程可用素材
导出场景深度需要时把场景深度数据导出，供后续流程使用

它不是替代所有旧工具，而是把「角色照明」这一段集中起来做。你仍然可以继续用旧的场景深度重建处理深度和遮挡；当你的重点变成照明、探针、角色融合或顺手导出深度时，再用本实验室。

怎么打开​

方式一：命令打开​

在游戏仓库根目录执行：

./dev.sh char-lighting

浏览器会自动打开角色照明实验室。

方式二：从 Web 控制台打开​

先打开 Web 控制台：

./dev.sh console

然后点 角色照明实验室。

方式三：指定端口且不自动打开浏览器​

如果你不想自动弹浏览器，或要固定端口给别人对照，可以这样开：

./dev.sh char-lighting -- --no-open --port 5311

入门：手把手做第一次​

第 1 步：选一个要调光的场景​

先选一个已经准备入库或已经入库的雾津场景，比如义庄门口、茶馆后巷、河滩淹尸处。你要先明确这次看的不是「背景图好不好看」，而是「角色站进去是否像在同一片光里」。

第 2 步：做伪世界重建​

让实验室读取场景图，生成或整理伪世界信息。你可以把这一步理解成给背景补一份「空间说明」：哪里近、哪里远、哪里可能挡住角色、哪里更容易接到光。

这一步的目标不是追求真实三维建模，而是够游戏使用：角色走进画面时，受光方向、阴影关系、前后层次能说得通。

第 3 步：放入角色看光照​

把一个常用角色放进场景里观察。建议用雾津里最容易暴露问题的角色：衣服暗、轮廓复杂、头发或帽檐边缘明显的角色。这样你更容易看出角色是不是发灰、过亮、像贴纸，或阴影方向和场景不一致。

第 4 步：调整角色光照​

围绕三个问题来调：

  1. 角色亮度是否和场景匹配。
  2. 角色阴影是否落在合理方向。
  3. 角色颜色是否融入雾津当场的冷暖气氛。

义庄夜景不要把角色打得像正午码头；茶馆灯下也别让角色脸部完全死黑。你要让玩家第一眼觉得「这个人确实站在这里」。

第 5 步：检查探针效果​

探针负责给角色采样局部光照。你可以在关键站位来回切换：门口、灯下、墙边、前景阴影旁。角色在不同位置的受光应该有变化，但不要跳得太突兀。

第 6 步：入库或导出​

满意后，根据当前目标选择：

  • 要让场景进入后续内容制作，就做场景入库。
  • 要给其它工具或流程继续用，就导出场景深度。
  • 要回游戏里看角色最终观感，就保存后再用 F5 进编辑器预览核对。

进阶：和旧「场景深度」工具怎么分工​

简单判断​

你要做什么推荐工具
修深度边缘、遮挡、碰撞场景深度重建
检查角色被桌子、柱子挡住是否正确场景深度重建
调角色在场景里的受光和阴影角色照明实验室
处理探针、局部光照、角色融入角色照明实验室
顺手导出场景深度给后续流程角色照明实验室

一句话：深度/遮挡仍可用场景深度；照明/探针/导出深度可用角色照明实验室。

典型流程​

一条常见流程是：先把场景图带进角色照明实验室，做伪世界重建；再放入角色检查光照与探针；满意后做场景入库，或按需要导出场景深度；最后回主编辑器用 F5 预览验证。

如果你已经在旧场景深度工具里把遮挡做得很好，不必因为开了本实验室就重做一遍。把它当成照明工作台：用现有深度作为基础，重点看角色融入和可导出的光照相关数据。

什么时候不要在这里硬调​

  • 角色穿过桌子、柱子、墙面：先回场景深度重建查遮挡和碰撞。
  • 背景图本身色调不对：先考虑滤镜工具或美术源图。
  • 只有全屏明暗要统一变化：未必需要角色照明实验室，滤镜或光照体积可能更直接。
  • 角色动画脚点不稳：去动画预览验动画，不要用灯光参数掩盖动作问题。

雾津例子：义庄门口夜班​

你要把「背尸-淹尸送义庄」交差点做得更可信：玩家把淹尸背到义庄门口，义庄灯笼从侧上方打下来，角色脸和肩膀有一点暖光，背后仍保持雾津夜里的冷暗。

  1. 打开角色照明实验室。
  2. 选义庄门口场景，做伪世界重建。
  3. 放入背尸时会出现的角色，站到义庄门口交差位置。
  4. 调整角色光照，让灯笼方向的受光更暖，背光面不要完全黑死。
  5. 在门口、台阶、墙边几个站位检查探针变化，确认角色不会忽亮忽暗。
  6. 场景资料满意后入库；如果后续流程需要，再导出场景深度。
  7. 回主编辑器 F5 预览，从河滩背尸到义庄交差，确认角色一路进入门口灯光时观感自然。

常见问题​

角色照明实验室是不是取代了场景深度重建？

不是。深度、遮挡、碰撞这类问题仍然可以用场景深度重建处理；角色照明实验室更适合处理角色受光、探针、场景入库和导出深度。

为什么我调完灯，角色还是穿桌？

这是遮挡或碰撞问题，不是照明问题。回场景深度重建修深度和碰撞，再回来检查光照。

导出的场景深度可以直接拿去用吗？

可以作为后续流程的数据来源，但你仍然要在游戏预览里检查实际遮挡、角色站位和光照效果。导出成功不等于场景最终观感已经过关。

固定端口什么时候用？

多人对照、录制教程、或者本机已经开了很多工具时，用 -- --no-open --port 5311 可以减少来回找页面的麻烦。
