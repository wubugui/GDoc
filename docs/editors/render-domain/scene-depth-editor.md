# 场景深度重建

等距场景若所有人像都贴在一张平板上，街就假了。**场景深度重建**从背景图估算**远近深度**，再手工修出碰撞区，让角色走过去时该被桌腿挡住、不能穿桌而过。这是主编辑器**场景面板的补充工具**——场景面板只留了两个深度相关的小旋钮，背景图本体、深度图、碰撞数据这些"大件"，主编辑器完全够不到，只能靠本工具生成、编辑、导出。

---

## 这是什么（30 秒看懂）

想象把一张茶馆内景的画剪成"立体剪纸"：近处的桌角、栏杆要"翘"在前面，远处的墙、柜台要"陷"进背景里。**深度图**就是记录这层前后关系的灰度图——游戏靠它判断"关二狗走到桌子后面时，半个身子该不该被桌子挡住"。**碰撞区**则更直接：告诉游戏"这一块地皮压根走不过去"，比如灶台、柜台内侧、供桌底座。

场景深度重建工具把这两件事放在同一个 3D/2D 混合视图里做：先自动估算深度，再用画笔手工修边缘，最后圈出碰撞区，一起导出给游戏用。它不管场景挂哪张背景、不管碰撞容差这类"小旋钮"——那些仍在主编辑器**场景**面板里改；它只管"深度图和碰撞这两大块数据从哪来"。

---

## 入门：手把手做第一次

1. 确认目标背景图已经**入库**（用[分类导入](../asset-domain/asset-ingest)），并且主编辑器**场景**面板已经把这张图挂在你要处理的场景上——比如"满堂茶客"茶馆内景。
2. 打开主编辑器（`./dev.sh editor`），菜单走 **工具 → 外部工具（新进程） → Scene Depth Editor**。场景深度重建没有自己的独立启动命令，只能从主编辑器里开。
3. 工具窗口里**打开选中场景**（或直接**导入背景图**），左侧能看到原图，右侧是**深度编辑**视图，可以切到**打开 2D 遮挡预览**看效果、也能切到**正交相机**的 3D 视图，用**方位角 / 仰角**转着看深度做得对不对。
4. 点**计算深度图**，工具会自动跑一次深度估计（界面上会显示"正在计算深度图..."）。
5. 对照原图检查深度：前景的桌沿、栏杆、立柱应该"更近"（发白），远处墙面、天花板应该"更远"（发黑）。用**笔刷**在深度图上直接画：**增大深度 / 减小深度 / 平滑 / 擦除**四种笔刷模式，配合**笔刷半径**和**编辑强度**滑条，把估算错的边缘修顺。
6. 切到**碰撞编辑**模式，用**多边形**工具在走不过去的地方画出碰撞区（柜台、灶台、供桌底），尽量和深度图里"前景"的位置对齐，同时留出过道。碰撞区可以调**碰撞图透明度**方便对照原图看位置准不准。
7. 点**导出游戏数据**，工具会把深度图和碰撞一起写到工程约定的位置。
8. 回主编辑器，**场景**面板刷新引用，用 F5 运行预览走进这个场景，验证关二狗从过道走过、被桌子挡住时是半身藏在桌后，而不是穿模飘在空中。

```mermaid
flowchart LR
  BG[背景图入库] --> OPEN[打开场景深度重建]
  OPEN --> EST[计算深度图]
  EST --> BRUSH[笔刷修深度<br/>加深/减浅/平滑/擦除]
  BRUSH --> COL[碰撞编辑<br/>画多边形]
  COL --> EXPORT[导出游戏数据]
  EXPORT --> SC[场景面板 引用]
  SC --> PV[F5 走位验证]
```

## 进阶：把每一项都讲透

**深度估算相关**
- **计算深度图**：对当前背景图跑一次自动深度估计，得到一张初始的远近灰度图，作为后续手工修整的起点。
- **拉伸检测阈值**：自动标记深度图里"被拉伸变形"的可疑区域（常见于透视夸张的柱子、栏杆），阈值越低越敏感，可以帮你少漏改一块该修的地方。
- **拟合非平面地面并预览**：场景地面如果不是纯平（比如有台阶、坡地），可以让工具尝试拟合出一个非平面地面并预览效果，减少手动一点点抠地面高度的工作量。
- **重置为原始深度**：改乱了想回到刚估算出来的那版深度，不用重新点计算，直接重置。

**笔刷修整**
- **增大深度 / 减小深度**：把画笔划过的区域整体调"远"或调"近"，用于修一整片被估错的墙面或桌面。
- **平滑**：抹掉深度图上生硬的锯齿或台阶感，让遮挡边缘看起来更自然。
- **擦除**：把某块区域深度清空，交给自动估算或后续手工重画。
- **笔刷半径 / 编辑强度**：半径决定一笔画多大范围，强度决定每一笔改变深度的幅度；处理细长的栏杆、桌腿建议半径小、强度低，慢慢描。

**3D/2D 预览与查看**
- **正交相机**：用无透视畸变的视角看深度图，配合**方位角、仰角**两个角度滑条可以像转小地球一样绕着看，检查深度起伏是否合理。
- **显示网格 / 显示坐标轴 / 显示刻度数字**：给 3D 视图加参照系，方便判断深度数值的实际尺度。
- **线框模式**：只看深度形成的网格骨架，不看贴图，专门用来检查有没有诡异的凸起或凹陷。
- **打开 2D 遮挡预览**：切回平面视角，直接看深度驱动出来的前后遮挡效果，这是最接近游戏里实际观感的预览方式。
- **像素/单位、移动步长**：控制视图缩放与移动的手感，场景大就调大步长，精修小物件就调小。

**Billboard（立牌）模式**
- **启用 Billboard (3D) / 加载 Billboard 贴图 / 使用默认贴图**：给场景里某些"立起来的薄片"物件（比如灯笼、旗子、单独一根柱子）挂一张始终朝向相机的贴图，让它们在深度视图里表现得像个简化的 3D 小立牌，不用真的建模。没有自定义贴图时，工具会用系统默认贴图占位。

**离地高度与阴影**
- **离地高度上限（着色）**：控制深度视图里"多高算是离地很远"的着色范围，主要影响可视化观感，帮你更快判断哪里是屋顶、哪里是贴地物件。

**碰撞编辑**
- **碰撞高度偏移**：整体调整碰撞区相对深度前景的对齐高度，碰撞区飘在物体上方或陷进物体里时用它统一纠偏。
- **生成碰撞草稿**：让工具按深度图自动猜一版碰撞轮廓（提示会显示"碰撞草稿已生成，可使用工具编辑"），你在这个草稿基础上增删调整，比从零画快很多。
- **碰撞图透明度**：半透明叠加碰撞区和原图，方便对照哪块该收窄、哪块该放宽。
- **加载碰撞 / 保存碰撞**：可以单独加载或保存碰撞数据，不必每次都连同深度图一起重新导出。

**存档相关**
- **新建场景 / 打开选中场景 / 保存场景 / 保存当前**：工具自己维护一份场景列表和当前编辑状态，和主编辑器的"场景"概念是同一批场景 id，但保存动作是各自独立的——工具里的保存是存这份深度/碰撞工作数据，不等于主编辑器的 Apply。
- **标定 Presets / 保存 Preset**：把一组常用的相机、笔刷参数存成预设，处理同类场景（比如同一批室内茶馆场景）时可以直接套用，不用每次都重新调视角和笔刷手感。

**和别的工具/面板配合**
- 深度图做完之后，是[光照体积烘焙](./lightvolume-lab)的直接输入——光源要贴着深度走，先把深度做对再去烘光，否则光斑会飘在半空。
- 场景面板里能改的只有深度配置的容差与地板偏移这两个数字，其余（背景层本体、深度图、碰撞主体）都要回本工具改，改完记得导出并让场景面板刷新。
- 视差场景（[视差场景编辑器](./parallax-editor)）和本工具无关——视差处理的是过场里的分层背景运动，不涉及玩家实际能不能走过去。

## 什么时候用它 / 和别的工具配合

| 情况 | 建议 |
|---|---|
| 新场景背景刚画完 | 深度 + 碰撞一次做完，再回场景面板摆 NPC、热区 |
| 角色穿桌、穿柱、穿墙 | 回本工具检查深度边缘和碰撞多边形，八成是某处碰撞漏了或深度前后判反了 |
| 只想改一点"容差"或"地板偏移" | 主编辑器场景面板就够用，不必开重的工具 |
| 换了一张新背景图 | 必须重新计算深度，旧场景的深度图不能直接复用到新图上 |
| 要做氛围光（灯笼、烛火局部提亮） | 先在这里把深度做扎实，再去[光照体积烘焙](./lightvolume-lab) |

**边界与当心**
- 深度估错边缘：挡人位置错位，看起来像角色飘在桌上而不是被桌子挡住。
- 碰撞画太大：角色离桌子还有一段距离就被挡住，走位显得很怪。
- 碰撞画太小：仍然能穿模钻进物体里。
- 忘记导出就回主编辑器 F5：预览用的还是旧数据，改了等于没改。
- 本工具管的只是深度和碰撞，全屏色调交给[滤镜工具](./filter-tool)，两者互不影响，别混着找问题。

## 常见问题

**Q：为什么场景面板里找不到"深度图"这个字段？**
A：背景层主体和深度图数据本来就不在主编辑器可编辑范围内，场景面板只留了depth_tolerance（深度容差）和floor_offset（地板偏移）两个小参数，其余必须在本工具里生成和导出，详见[危险区](../concepts/danger-zone)。

**Q：我改了深度，F5 进去怎么还是老样子？**
A：先检查有没有点"导出游戏数据"——工具里的修改在导出前只存在于工具自己的工作状态里，没导出就相当于白改。

**Q：碰撞草稿自动生成的轮廓不太准，要从头画吗？**
A：不用，草稿只是起点，在它基础上用多边形工具增删调整比从零画快很多。

**Q：Billboard 贴图是干什么用的，什么时候该用？**
A：给灯笼、旗子这类"薄片朝向相机"的小物件用的简化处理，不需要真的建模出前后深度，普通的桌椅、墙面不需要它。

**Q：新背景图能沿用旧场景的深度图吗？**
A：不能。换了背景图必须重新跑一次计算深度图，旧的深度图是按旧图像素对齐的，直接复用只会全错位。
