# 边改边看：运行预览

改雾津的内容，最怕「册子上写得挺美，进游戏却不对」。运行预览让你在主编辑器里**直接跑游戏**——左边改，右边看，不用来回切窗口。这一页教你怎么打开它、三种运行方式怎么选、以及日常编纂应该养成的验证节奏。

## 这是什么（30 秒看懂）

**大白话：** 运行预览就是把游戏**嵌进**主编辑器窗口里的一块画面。你不用开另一个浏览器标签页去玩，改完东西直接在编辑器里按一下键，游戏就在旁边跑起来，走到对应场景/对话就能看到你刚改的效果是不是对的。它和你在浏览器里单独跑起来的游戏是**同一套游戏、同一份数据**，只是显示的位置不同。

## 入门：手把手打开一次

1. 启动主编辑器：`./dev.sh editor`
2. 左侧导航树滚到最底下，展开 **运行与预览**
3. 点 **Game**

右边编辑区会变成一块**内嵌游戏画面**——跟你在浏览器里跑的是同一套游戏，只是嵌在主编辑器窗口里。

一键运行的三个快捷键：

| 操作 | 快捷键 | 发生什么 |
|---|---|---|
| **运行（内嵌）** | `F5` | 先自动保存，再启动游戏，画面嵌在 Game 页里 |
| **独立窗口预览** | `Ctrl+F5` | 另开一个窗口跑游戏，方便双屏对照 |
| **停止** | `Shift+F5` | 关掉正在跑的游戏进程 |

:::tip[推荐工作流]
改一句对白 → `Ctrl+S` 保存 → `F5` 运行 → 在游戏里走到关二狗面前触发对话 → 不对就 `Alt+←` 回图对话继续改。这套循环就是日常编纂节奏。
:::

## 进阶：把这套验证节奏用透

### 完整走一圈：改城隍庙对白举例

```mermaid
flowchart LR
    A[图对话里改台词] --> B[Ctrl+S 保存]
    B --> C[F5 运行预览]
    C --> D[走进城隍庙触发对白]
    D --> E{效果对吗?}
    E -->|不对| A
    E -->|对了| F[继续编下一段]
```

1. **叙事编排 → 图对话**：找到关二狗相关对白图，改 `line` 节点的文字。
2. **`Ctrl+S`**：保存图对话面板。
3. **`F5`**：编辑器先存盘，再拉起游戏。切到左侧 **运行与预览 → Game** 看画面。
4. 用键盘或鼠标操作角色，走到触发对白的 **[热区](../../reference/glossary)**，确认文字、立绘、选项都对了。
5. 不对就 `Shift+F5` 停掉，`Alt+←` 回到图对话继续改，再 `F5`。

### 内嵌预览 vs 独立窗口 vs 浏览器

三种看法，按场景选：

| 方式 | 怎么开 | 什么时候用 |
|---|---|---|
| **内嵌预览** | `F5`，看 Game 页 | 日常最顺手，改完立刻看，不用切窗口 |
| **独立窗口** | `Ctrl+F5` | 双屏：一屏编辑器、一屏游戏；或者想要更大的游戏画面 |
| **浏览器里跑** | Web 控制台的 **游戏 dev 开关**，或 `./dev.sh game start` | 已经开着游戏、不想重启编辑器时；或想用浏览器自带的开发者工具排查 |

内嵌预览、独立窗口、浏览器里跑的游戏，本质上都是**同一份游戏数据**。你在主编辑器里保存后，不管走哪种方式验证，读到的都是最新内容——三者的区别只在「画面显示在哪」，不在「数据是不是同一份」。想统一起来管理起停，见 **[Web 控制台](../concepts/web-console)**。

### 为什么改内容要靠运行预览验证，而不是只看编辑器界面

编辑器面板显示的是「你填的参数」，不是「玩家实际会看到的效果」——条件是否真的成立、动作是否真的按预期顺序执行、富文本标签是否正确解析成名字，这些都得进游戏里实际触发一遍才能确认。尤其是：

- **条件类改动**：改完条件，至少要在「满足」和「不满足」两种状态下各运行一次，确认分支符合预期，见 **[怎么设条件](../concepts/conditions)**。
- **动作类改动**：多条动作串起来的效果（先给物再改旗标再弹提示），最好完整走一遍，确认顺序和参数都对，见 **[怎么编排动作](../concepts/actions)**。
- **富文本类改动**：标签解析对不对、图片短名是否存在，编辑器不一定当场报错，得进预览里亲眼看到显示结果，见 **[怎么写带引用的文本](../concepts/rich-text)**。

### 反复运行时的节奏建议

- 改一个小点就 `F5` 一次，不要攒一堆改动最后才验证——出问题时更容易定位是哪一步改坏的。
- 牵扯到「重建区」相关的编辑（比如打开过的图对话节点、热区详情），保存后先确认面板里内容还完整，再 `F5` 进游戏看效果，两步都不能省，见 **[危险区](../concepts/danger-zone)**。
- 独立窗口（`Ctrl+F5`）适合验证耗时长、需要边看编辑器边操作游戏的场景；内嵌预览适合快速反复的小改动。

## 危险区与边界

- `F5` 会自动帮你保存**当前停留的这块面板**，但不会替你保存其它没切回去看过的面板——多面板改动务必自己 `Ctrl+Shift+S` 全存一遍，再运行预览，否则可能验证到的是半新半旧的状态。
- 运行预览读到的是**已保存**的工程数据，不是你界面上还没保存的临时改动——预览里看到的和你屏幕上正在编辑的内容不一致时，第一反应应该是「是不是忘了保存」。
- 内嵌预览、独立窗口、浏览器版本都指向同一份数据，但各自是**独立的运行会话**——不要指望三边画面自动互相同步，每次想看最新效果，都需要重新触发一次运行或刷新。

## 常见问题

**按了 F5 没反应？**

- 先确认左侧选的是 **运行与预览 → Game**，或等游戏进程在状态栏显示已启动。
- 若提示环境未就绪，回游戏仓库根目录跑一遍 `./bootstrap.sh`。

**改了内容，游戏里还是旧的？**

- 确认对应面板已 `Ctrl+S` 保存。`F5` 会再存一遍，但别的面板若没存，那些改动不会进游戏。
- 停掉预览（`Shift+F5`）再 `F5` 重开。

**预览里操作卡顿？**

- 开发服第一次启动要几秒，属正常。复杂场景可先用独立窗口（`Ctrl+F5`）减轻编辑器负担。

**内嵌预览和独立窗口能不能同时开着？**

两者都是各自的运行会话，同时开着容易分不清哪个是最新状态、也更占资源。建议同一时间只留一个在跑，需要切换方式时先 `Shift+F5` 停掉当前的，再用另一种方式重新运行。

**运行预览里角色乱走、事件没触发，是不是我东西改坏了？**

先排除是不是保存没做全——切回对应面板确认状态栏显示已保存。确认保存无误后仍然不对，再按 **[出问题怎么办](../../tutorials/troubleshooting)** 的思路排查条件、动作是不是哪里配置有误。

**每次验证都要重新走一遍完整流程很麻烦，有更快的办法吗？**

如果编辑器所在项目配了 Web 控制台的场景快捷入口或叙事跳转，可以直接跳到你要验证的那个场景/叙事点位，不用从头玩，见 **[Web 控制台](../concepts/web-console)**。
