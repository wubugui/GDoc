怎么写带引用的文本

雾津册子里的句子很少写死。「你」可能是玩家化身名，「那规矩」可能随进度变文案，物品名要对上背包里的登记——这类 带引用的文本 在主编辑器里到处出现：对白、物品描述、档案正文、任务说明、规矩条文。这一页教你用 [标签:…] 写法，让运行时自动换成对的词；也讲清哪些地方能插图、哪些地方不能。

这是什么（30 秒看懂）​

大白话： 在句子中插一段特殊标记，游戏播放时换成当前对应的名字、数值或链接，而不是你手打的死字。

例如档案里写：

据说 [物品:狗哨] 能引开雾津巷口的野狗——[NPC:关二狗] 从不承认他丢过。

玩家读到时，「狗哨」「关二狗」会从游戏里登记的名称解析出来；你改了物品的显示名，全文自动跟着变，不用一句句手改。

入门：手把手做第一次​

以「把关二狗一句台词里的『狗哨』改成随物品名变化的引用」为例：

  1. 打开 图对话 → 选中关二狗的台词节点。
  2. 在 台词 富文本框里定位到「狗哨」这两个字准备替换。
  3. 点框边的 插入 按钮 → 选 物品 → 在列表里选「狗哨」。
  4. 框里会出现一段引用标签，前后补齐对白原文，删掉原来手打的「狗哨」两个字。
  5. Ctrl+S 保存，F5 运行预览，走一遍对话，确认显示的名字正确。
雾津小例子

以后你把「狗哨」的显示名改成「褪色的狗哨」，图对话里这句台词不用回来改，运行时会自动显示新名字——这就是引用标签的意义。

进阶：每一项都讲透​

八种常用引用标签​

输入格式统一为 [类型:标识]（界面里常有 插入 按钮帮你生成，不必死背语法）：

标签引用什么用法说明例子
[string:…]文本库里登记的固定条目适合系统统一用语、多处要保持一字不差的措辞；改文本库里的一条，所有引用处跟着变系统提示、格式化短语
[flag:…]某个旗标的当前值把旗标本身当文字显示出来，适合「进度到哪儿文案就变到哪儿」的场合显示好感等级、计数进度
[item:…]物品的显示名玩家读到的永远是物品当前登记的名字[item:狗哨] → 狗哨
[npc:…]角色的显示名同一个角色若在多个场景各有一份实例，可以在标签里加一个场景限定，指明具体引用哪一份，避免叫错人[npc:关二狗] → 关二狗
[player:…]玩家相关的信息化身名、称谓等，随玩家当前设置变化「[player:name]，回来啦？」
[quest:…]任务标题任务改名后引用处自动更新「关于 [quest:寻狗记] 的线索」
[rule:…]规矩的条文摘要引用规矩当前登记的文案层「违反 [rule:城规三则] 者……」
[scene:…]场景显示名场景改名后引用处自动更新「前往 [scene:雾津码头]」

完整语法边界与更多例句见 富文本标签速查。

在哪会遇到富文本框​

面板典型字段
图对话台词、选项文案
物品描述、动态描述
档案条目正文（可插图）
任务标题、任务说明
规矩各层正文
遭遇选项与结果文字
文本库字符串值
场景热区显示标签

框边若有 插入引用 按钮，点选类型和条目即可插入，比手打不易错——优先用按钮，别从别处复制粘贴 id 手打。

插图标签 [img:…] 的特殊规矩​

正文里还能插 [img:短名] 显示一张图片，但这条标签的待遇和其它八种不一样：

  • 只有 档案 面板（人物簿/见闻录/杂书匣/书籍内容）提供插图的「插入」按钮，点选就能生成，最省心。
  • 其它任何位置（场景热区标签、图对话台词、物品描述等）没有这个按钮，得自己手打 [img:短名]。手打的标签不会丢——它就是普通文本，跟着这个字段正常保存；但编辑器不会帮你校验这个短名是否真的存在，打错字或忘了登记，运行时才会发现引用不到。
  • 图片短名要先在 叠图 面板或对应资源流程里登记好，标签才引用得到东西——这一步别漏，不然标签本身没问题，也找不到对应的图。
雾津实例

雾津见闻录里要配一张城隍庙线图，优先在 档案 编辑器里用插图按钮插入，最省心也最不容易打错短名。其它地方要插图（比如场景热区说明）就得手打 [img:短名]——这只是「没有便捷按钮」，标签本身写对了、短名在叠图里登记过，照样能正常保存和显示。

哪些字段不接受标签，只能写死字​

以下多是纯标识或只读引用，界面通常是普通单行文本框，不可以插引用标签：

  • 各种内部 id 字段（任务 id、场景 id、物品 id 等）——这些本来就是给系统识别用的标识符，不是给玩家看的文案。
  • 规矩碎片上「所属规矩」这类只读引用行——它本身就是引用，不需要再套一层标签。

判断标准很简单：玩家会读到的文案（对白、描述、任务说明、规矩正文、文本库字符串值）走的是富文本框，可以插标签；系统内部用来对应/查找的标识是纯文本框，不能插也不需要插。

和动作、条件的关系​

  • 富文本只管「显示什么字」。
  • 条件 管「要不要显示某选项 / 能不能进」。
  • 动作 管「这句话说完之后发生什么」。

一句对白可以只用到富文本；这个节点会不会出现，由图上的条件或分支决定；对白之后要不要给东西，由挂在节点上的动作决定。三者各司其职，别混着往一个框里塞。

危险区与边界​

  • 插图的「插入」按钮只有档案面板才有；别处手打的 [img:…] 只是没有便捷按钮可用，编辑器不会校验短名是否存在，但标签本身作为文本会正常保存，不会丢——见 危险区。
  • 富文本框所在的父条目不用担心手写内容被连带清掉：场景热区的详情、图对话节点这些字段都是往返无损，编辑器会保留它不认识的内容。
  • 标签里的标识符（物品 id、NPC id 等）必须和游戏里登记的完全一致，编辑器不一定当场报错——错的标识多半表现为显示空白或退回默认文案，而不是弹窗提示。

常见问题​

标签显示成空白或者原样显示方括号，怎么回事？

多半是标签里的标识符打错了，或者和登记的名字对不上（包括大小写、多余空格）。优先用 插入 按钮生成标签而不是手打，可以避免这类问题。

一句话里能不能叠好几个标签？

可以，一句对白里插多个不同类型的标签没有问题，例如同时引用 NPC 名和物品名。

[npc:…] 里的场景限定是干什么用的？

同一个角色如果在不同场景各摆了一份实例（比如关二狗在城隍庙和码头都有出场），场景限定用来告诉运行时具体引用哪一份，避免显示成另一个场景里的状态。普通情况下角色只有一份实例，不需要额外填这个。

为什么插图按钮只有档案面板才有？

档案是唯一把插图当作一等公民对待的编辑面——面板专门给插图配了浏览、选择、插入的完整流程。其它面板的文本框设计初衷是纯文案，插图属于「能用但没人管」的边界用法，所以没有配插入按钮。

文本库标签 [string:…] 和直接手打文字有什么区别？

手打的字是死的，改一处只影响这一处。[string:…] 引用的是文本库里登记的同一条内容，多处引用同一个 [string:…] 时，改文本库那一条，所有引用处一起变——适合需要保持统一措辞的系统用语。

旗标标签 [flag:…] 显示的是什么？

显示的是这个旗标当前的值，作为文字直接嵌进句子里——适合「随进度变化的称谓、计数」这类场合，和条件里查旗标是否满足某个比较式是两回事，别混淆。
