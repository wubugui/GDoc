# 编年史 v2

若你想让多个"角色 Agent"在雾津设定里**按周互动、自己写编年史、落盘成档案**，而不是你手改每一条任务台词——**编年史 v2** 提供一个**桌面 GUI**，也支持**命令行按周批量跑**。数据以"一次实验一个文件夹"的方式存放，适合叙事实验、世界观压力测试、生产管线之外的沙盒推演。

:::info[和 v3 别混]
`./dev.sh chronicle-sim-v2` 打开的是 **v2**（本页）；`./dev.sh chronicle-sim`（不带版本号）打开的是 **v3**（[编年史 v3](./chronicle-sim-v3)）。控制台上这也是两个分开的按钮："编年史 v2" 和 "编年史 v3"，点错会进到完全不同的界面。
:::

---

## 这是什么（30 秒看懂）

把它想象成一个"给雾津开的沙盘推演桌"：你先摆好几个角色（比如关二狗、码头帮会、庙祝）、给他们一点世界观提示，工具会一周一周地推进——每周由背后的语言模型驱动这些角色"活"一遍，产生事件、谣言、关系变化，全部写进这一次实验专属的档案文件夹里。你可以像翻剧本草稿一样浏览这些产出，挑出写得好的桥段，再动手迁进主编辑器变成正式内容。

它**不是**主编辑器的替代品——沙盘推演出来的东西终究是"灵感草稿"，不会自动变成玩家能玩到的任务或对话，正式内容永远要人工搬回主编辑器。它也不负责游戏正式运行时读什么数据，纯粹是内容生产前期的实验场。

## 入门：手把手做第一次

1. 确认本机已经装好项目 Python 环境，并且已经装了驱动多 Agent 对话所需要的命令行工具链（首次使用请找项目负责人确认依赖是否齐全，没装好会导致"周模拟"跑不起来）。
2. 命令行跑 `./dev.sh chronicle-sim-v2` 打开主窗口；也可以从 Web 控制台点"编年史 v2"按钮。
3. 主窗口有四个标签页：**模拟**（跑周次推演）、**种子编辑**（准备角色和世界观设定）、**编年史**（浏览已产出的档案内容）、**设定库**（管理可复用的灵感条目）。
4. 先去**种子编辑**标签，**新建 Run**（一次实验对应一个独立的运行目录），可以从**设定库生成**种子，或者直接粘贴/编辑角色种子内容——种子里写清楚参与的角色、背景设定、性格标签这些信息。工具自带一个演示模板可以参考，模板本身就是取材自雾津设定的示例（一批民国川渝风的人物和地点，比如朝天门码头、望江茶馆），可以照着它的写法调整成你要的雾津角色组合。**写入世界**前工具会做一次解析校验，有问题会在"发现以下问题"里列出来，可以选择直接确认忽略或者先改好再写入。
5. 切到**模拟**标签，选中刚建的 Run，配置好这一周要跑的角色范围，点**运行本周**（或界面上等价的按钮）。工具会调后台驱动每个角色对这一周发生了什么"表态"，产生事件、对话片段、谣言传播。
6. 切到**编年史**标签，浏览这一周的**世界**总览、**事件**列表、**谣言**传播情况；可以对某条内容点**总结**看归纳、点**意图**看角色为什么这么做，也能**搜索编年史内容**、给会话**清空**重来。
7. 挑出觉得有意思的桥段，人工**摘选**进主编辑器——比如把某周产出的冲突写成任务梗概，回到主编辑器**任务**面板手动新建一条正式任务。档案本身留在运行目录里，不会自动删除，留作策划会对照参考。

```mermaid
flowchart LR
  SEED[种子编辑: 角色/世界观] --> RUN[新建 Run]
  RUN --> SIM[模拟标签: 运行本周]
  SIM --> ARCH[编年史标签: 浏览事件/谣言/总结]
  ARCH --> PICK[人工摘选精彩桥段]
  PICK --> ED[主编辑器 正式内容]
```

## 进阶：把每一项都讲透

**四个标签页**
- **模拟**：实际跑周次推演的地方，可以运行单周，也可以一次跑一个范围（比如连续跑第 1 到第 3 周），跑的过程中可以**取消模拟**中断。
- **种子编辑**：准备"这次实验有哪些角色、他们是什么设定"的地方。角色种子包含名称、性格标签（比如豪爽仗义、精明干练、圆滑世故、憨厚老实这类描述）、所属阵营/组织、初始**生命状态**。写好后点**保存 Tier**、**写入世界**落盘到当前 Run。
- **编年史**：浏览产出内容的地方，支持关键词**语义搜索**，找不到相关结果时会有对应提示；每条内容附带来源草案类型/ID，方便追溯这条内容具体是哪个角色在哪个环节产生的。内容看多了想清空重新读，可以**清空会话**。
- **设定库**：维护一批可复用的"灵感条目"——标题、内容、标签（逗号分隔）、来源，方便你把之前实验中好用的世界观设定、角色小传攒起来，下次新建 Run 时直接**从设定库生成**种子，而不用每次从零写。支持**导入 MD 文件**批量导入已有的设定文档。

**LLM 配置（决定"谁在替角色说话"）**
- 工具把不同的推演环节拆给不同角色定位的语言模型调用槽位，常见的角色定位包括：**编年史导演**（统筹整体节奏走向）、**周总结/月史**（归纳阶段性摘要）、**谣言**（模拟谣言传播与走样）、**风格重写**（把内容改写成贴合雾津的语气）、**语义记忆**（做长期记忆的语义检索）、**种子抽取**（从设定库文本里提炼结构化种子）、**探针问答**（临时追问某个角色"你当时怎么想的"）。
- 每个槽位可以单独配置**提供方**、**模型名**、调用**方式**，并且有**测试本槽连接**、**测试嵌入**两个按钮，用来在正式跑周次之前先确认这个槽位配置是通的，避免跑到一半才发现某个角色没法说话。
- **单条 trace 最大字符数**、**输出 new_messages / all_messages 大段 JSON** 这类是调试与排障用的开关，日常摆内容不需要碰，出问题找项目负责人一起看。

**谣言传播机制**
- 编年史里专门有一条**谣言**线索：角色之间口耳相传时,内容会"走样"——工具会标注某条传闻是**未走样**还是**走样**过的版本，方便你观察"一个事件传到第三个人嘴里变成什么样"这种市井小道消息的失真过程,这本身就很贴雾津的市井气质。
- 除了 GUI 里直接看,也可以用命令行单独触发一次谣言传播的校验跑,或者对某个已有 Run 的某一周单独统计谣言传播情况——这两个都是给要做批量分析、无人值守跑数据的同事用的,具体命令找项目负责人要,不建议凭记忆现拼。

**命令行周模拟（批量/无人值守）**
- 除了在 GUI 里点按钮,也可以直接用命令行推进某个 Run 的某一周:`./dev.sh chronicle-week <运行目录> --week <周次>`,GUI 里"运行本周/运行范围"背后调用的其实是同一套推演逻辑,所以命令行和 GUI 跑出来的结果口径是一致的。适合要连续无人值守跑一整季、或者接进自己脚本流水线的场景。

**和别的工具/面板配合**
- 沙盘产出的桥段绝不能直接当定稿——必须经过策划审核,人工摘选进[主编辑器](../main-editor/overview)的**任务**或[叙事状态机](../narrative-domain/narrative-editor-web)才算数。
- 想要"图驱动、纯命令行、可脚本化"的推演方式而不是这种"桌面点按钮+Agent 写作文"的路线,看[编年史 v3](./chronicle-sim-v3)。
- 和游戏正式运行的数据完全隔离,改工程正式内容还是要回主编辑器,本工具的运行目录不会被游戏读取。

## 什么时候用它 / 和别的工具配合

| 情况 | 建议 |
|---|---|
| 试"若纸人势力三周后控制码头"一类假设 | 用 v2 沙盘按周推演,看看会不会自发长出可用的冲突桥段 |
| 正式改寻狗记主线内容 | 回[主编辑器](../main-editor/overview)叙事图 + 任务面板 |
| 想要图驱动、纯 CLI、可批量脚本化的推演 | 考虑[编年史 v3](./chronicle-sim-v3) |
| 本机没装 Agent 命令行工具链 | 先找项目负责人确认依赖装好,否则"运行本周"跑不起来 |
| 需要连续无人值守跑一整季 | 用命令行 `./dev.sh chronicle-week` 而不是逐周点 GUI |

**边界与当心**
- 别把 v2 和 v3 的命令搞混,见页首 info。
- 沙盘产出的内容不是 canon(定稿),必须经过策划审核才能进正式工程数据。
- 运行目录一旦手动删除,这次实验的档案不可恢复。
- 语言模型产出的内容会有"幻觉"(编得像真的一样但其实不对),当灵感来源看待,不要直接照抄当最终台本。

## 常见问题

**Q：种子编辑里那些"关二狗""朝天门码头"是雾津正式设定吗？**
A：这是工具自带的演示模板,取材自雾津本身的世界观风格,可以直接照着它的结构改成你要的角色和地点组合,但模板本身的具体人物关系、地点细节仅供参考,不代表任务面板里的正式内容。

**Q：跑"运行本周"没反应或报错，是哪里的问题？**
A：先检查本机是否装好了驱动多 Agent 对话的命令行工具链和对应的账号凭据,这是最常见的卡点;其次去 LLM 配置里对相关槽位点"测试本槽连接"确认能通。

**Q：档案库里的内容能直接复制进任务面板吗？**
A：不建议直接复制粘贴——沙盘产出的语言风格和结构未必符合正式内容的格式要求,应该提炼成梗概,再在主编辑器里手动新建正式条目。

**Q：谣言"走样"是什么意思，对我有什么用？**
A：模拟"小道消息传几手就变味"的过程,可以帮你观察一个事件在市井间口耳相传会怎么失真,给设计"传闻类"内容提供灵感,不是游戏正式的谣言系统。

**Q：能不能不开 GUI，只用命令行批量跑一整季？**
A：可以,用 `./dev.sh chronicle-week <运行目录> --week <周次>` 逐周跑,或者按范围连续跑,背后逻辑和 GUI 里点"运行本周/运行范围"是同一套,结果口径一致。
