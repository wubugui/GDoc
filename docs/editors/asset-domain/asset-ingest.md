# 分类导入

你从画师、音效师或 AI 管线手里拿到文件，还不能直接填进场景——得先按工程约定**分类搬进正确文件夹**。**分类导入**就是这道「入库」工序：选外部文件、选目标类别、确认落位，避免手拖文件夹搞乱结构、放错地方导致游戏里加载不到。

入库完成后，用 [资源浏览器](./asset-browser) 核对；再在主编辑器各面板里引用。

---

## 这是什么（30 秒看懂）

把它想成雾津货栈的**收货口**：画师、音效师、AI 管线送来的箱子（图片、视频、音频文件）先在这里过一道秤，按「这是立绘」「这是背景」「这是音效」分好类，才能进对应的仓（工程目录）；不走这道口子直接把文件手动拖进工程文件夹，容易放错位置——游戏是按固定目录结构去找资源的，放错目录，主编辑器面板里就会「找不到」。

分类导入**只管落位**，不改文件内容：不做缩放、不抠像、不抽帧，那些是 [图片缩放](./image-resizer)、[视频转图集](./video-to-atlas) 等专项工具接下来的活；它也**不会**帮你在场景、角色里接上引用——文件进了工程之后，回主编辑器对应字段手动选中它，才算真正“用上”。

## 入门：手把手做第一次

**场景**：画师刚发来一张城隍庙夜祭的背景图，还有一份关二狗新立绘，你要把它们分别归位。

1. 打开分类导入窗口（见下方「怎么开」）。
2. 点 **添加文件…** 或直接把文件（也可以是整个文件夹）拖进窗口中间的列表——文件夹会自动展开成里面所有的图片/视频/音频文件。
3. 在「分类」下拉里，为城隍庙背景选**游戏 / 背景**；关二狗立绘另建一条，选**游戏 / 立绘**。
4. 立绘这类分类支持**子目录**——比如在子目录框里填「关二狗」，这批立绘就会归到立绘目录下专属的关二狗子文件夹里，而不是和别的角色混在一起。
5. 窗口下方会实时预览**目标落点**（工程里的相对路径），确认没写错、没落到奇怪的地方。
6. 是否勾选**移动（否则复制）**：不勾就是复制一份进工程、原文件保留在原处；勾选则原文件直接搬进工程，原处不再留底。第一次操作建议不勾，留个原始文件保险。
7. 点**导入选中分类**，等下方日志显示完成。
8. 打开 [资源浏览器](./asset-browser)，按分类目录找到刚导入的文件，逐个预览确认。
9. 回主编辑器：城隍庙背景去 **场景** 面板背景字段选中它；关二狗立绘去 **[角色登记](../panels/character)** 绑定。

```mermaid
flowchart LR
  DL[下载夹 / 协作包] --> PICK[选文件或文件夹]
  PICK --> CAT[选分类 + 可选子目录]
  CAT --> PREVIEW[预览落点]
  PREVIEW --> RUN[执行导入]
  RUN --> BR[资源浏览器 核对]
  BR --> REG[主编辑器 登记引用]
  REG --> PV[F5 预览验证]
```

## 进阶：每一项都讲透

### 分类怎么选

| 分类 | 用来放什么 | 支不支持子目录 |
|---|---|---|
| 游戏 / 背景 | 场景背景图 | 不支持，一般直接放根下 |
| 游戏 / 立绘 | 角色立绘、道具插画 | 支持——填个子目录名（如角色名、道具类别），同类素材不会和别人混在一起 |
| 游戏 / 角色头像 | 角色头像小图 | 不支持 |
| 游戏 / 音频 | 音效、BGM | 不支持 |
| 工作区 / 动画原视频 | 还没处理的角色动作原始视频，等着送进[视频转图集](./video-to-atlas)或[动画后处理](./animation-pipeline) | 不支持 |
| 工作区 / 未分类收件箱 | 一时定不下类别的文件，先扔这里，之后再用[资源浏览器](./asset-browser)二次搬家整理 | 不支持 |
| 自定义相对路径 | 以上都不合适时，手填一个工程内的目标路径 | 路径必须以约定的两个根目录之一开头（游戏可加载的资源根，或工作区数据根），否则视为无效目标，不允许导入到工程外或任意位置 |

### 每一项控件干什么

| 控件 | 干什么 |
|---|---|
| 添加文件… | 弹出选择框，可多选文件；只收图片/视频/音频这类素材文件，无关文件类型会被自动过滤掉，不会误收杂七杂八的文档。 |
| 添加目录… | 选一整个文件夹，工具自动把里面所有符合类型的文件递归找出来加进列表。 |
| 拖入文件/文件夹 | 直接从系统文件管理器拖到列表里，效果和「添加」按钮一样。 |
| 清空列表 | 清空当前待导入列表，不影响已经导入过的文件。 |
| 移动（否则复制） | 勾选=从原位置**搬走**（原处不留底）；不勾=**复制**一份进工程，原文件保留。协作素材建议先用复制，留个原始底稿。 |
| 打开目标目录 | 直接跳到当前选中分类对应的工程目录，方便你不开浏览器就先看一眼。 |
| 落点预览 | 实时显示这批文件真正会落到工程里的哪个相对路径，导入前务必看一眼确认。 |
| 导入选中分类 | 真正执行——把列表里的文件按当前选的分类和子目录搬/复制进去。 |
| 操作日志 | 下方文本框实时滚动显示每个文件的导入结果（成功/失败），导入完成后也会弹窗汇总统计。 |

### 老手技巧

- **同名不会覆盖**：如果目标目录已经有同名文件，工具会自动在文件名后加数字后缀（如加 `_1`）避免覆盖，绝不会悄悄把你已经在用的成品图冲掉。但这也意味着——如果你是想「替换」旧版本，导入后目录里会同时有新旧两个文件，你还得自己去 [资源浏览器](./asset-browser) 删旧的，或者去主编辑器把引用改指向新文件，工具不会替你二选一。
- **每次导入都留痕**：每次执行导入都会追加一条可追溯的入库记录（时间、来源、落点），方便事后查「这批素材是什么时候进来的」；记录数量超过上限时只保留最近的一批，不会无限膨胀。
- **协作同事发来一整个 zip**：先解压到本地临时文件夹，再按类型分几批分别导入（背景一批、立绘一批、音效一批），别图省事整个文件夹一股脑选同一个分类，容易把音效和图混进同一目录。
- **超大原图先别急着降尺寸**：背景图可以先原样导入占位，等确定真的要用了，再走 [图片缩放](./image-resizer) 出一份适配游戏分辨率的副本，两步分开做更稳。

## 和其它工具的配合

| 工具 / 面板 | 关系 |
|---|---|
| [资源浏览器](./asset-browser) | 导入完成后的核对与后续整理（重命名、加标签、清理）都在这边做。 |
| [图片缩放/镜像](./image-resizer) | 尺寸、朝向不合适时，另开工具处理出副本，再把副本重新导入或直接在场景里引用。 |
| [视频转图集](./video-to-atlas) | 动画原始视频入库后的下一道工序：一段段视频抽帧拼图集。 |
| [动画后处理](./animation-pipeline) | 视频已经批量稳定化好的情况下，作为动画原视频入库后的另一条自动化路线。 |
| [主编辑器](../main-editor/overview) | 素材真正被使用的地方——场景、角色、物品等各面板里手动选中已入库的文件。 |

## 常见问题

**Q：分类选错了，文件已经导入到错误目录，怎么办？**
去 [资源浏览器](./asset-browser) 里用「移动到…」把文件挪到正确目录，或者重新走一次分类导入选对分类（记得把错放的那份清理掉，避免同一份素材散落两处）。

**Q：导入后主编辑器里还是找不到这张图？**
分类导入只负责把文件搬进工程，**不会**自动帮你在场景/角色/物品字段里接上引用——文件进去之后，还要回对应面板手动选中它一次。

**Q：同名文件导入会覆盖旧的吗？**
不会。工具检测到同名会自动加数字后缀另存一份，新旧文件都会保留在目录里；如果你的目的是「替换」，导入完还要自己去删旧文件或改引用。

**Q：勾了「移动」，原文件是不是就没了？**
是的，「移动」会把原文件真的搬进工程、原位置不再保留副本。不确定的话优先用默认的「复制」，需要清理原始文件夹时再手动删。

**Q：自定义相对路径填了却导不进去，提示无效？**
自定义路径必须落在工程约定的两个根目录之一下（游戏运行时会加载的资源根，或编辑器工作区数据根），随便填一个工程外的路径或工程内其它位置都会被判定无效，这是防止把文件散落到不该去的地方。

**Q：能一次导入很多不同类型的文件、自动分类吗？**
不能，一次导入操作只能应用一个分类目标；不同类型的文件（比如背景图和音效）建议分批选择对应分类分别导入。

## 怎么开

**方式一：命令**

```bash
./dev.sh asset-ingest
```

**方式二：Web 控制台**

```bash
./dev.sh console
```

点 **资源入库**（或控制台里与入库对应的按钮）。
