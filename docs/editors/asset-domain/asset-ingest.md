# 分类导入

你从画师、音效师或 AI 管线手里拿到文件，还不能直接填进场景——得先按工程约定**分类搬进正确文件夹**。**分类导入**就是这道「入库」工序：选外部文件、选目标类别、确认落位，避免手拖文件夹搞乱结构。

入库完成后，用 [资源浏览器](./asset-browser) 核对；再在主编辑器各面板里引用。

---

## 干什么

- 把**工程外**素材按类别导入既定目录。
- 按类型分流：背景、立绘、音效、UI 等（以工具里列出的类别为准）。
- 记录入库操作，方便事后追溯「这批图什么时候进来的」。

本工具**不**做缩放、抠像、抽帧——那些走 [图片缩放](./image-resizer)、[视频转图集](./video-to-atlas) 等专项流程。

---

## 怎么开

**方式一：命令**

```bash
./dev.sh asset-ingest
```

**方式二：Web 控制台**

```bash
./dev.sh console
```

点 **资源入库**（或控制台里与入库对应的按钮）。

---

## 一步步怎么用

1. 打开分类导入窗口。
2. 点 **添加文件** 或拖入外部素材（可多选）。
3. 为每批素材选**目标类别**——例如环境背景、角色立绘、音效。
4. 预览目标路径与命名规则，确认不会覆盖已有成品。
5. 执行导入，等进度完成。
6. 打开 [资源浏览器](./asset-browser)，逐文件确认落位与预览正常。
7. 回主编辑器在对应面板引用新资源。

---

## 何时用

| 情况 | 建议 |
|---|---|
| 新收到一批美术/音频 | 第一时间分类导入，别散落在下载夹 |
| 替换旧版立绘 | 导入新文件后，在主编辑器改引用或同名覆盖策略 |
| 协作同事发来 zip | 解压后分批导入，每批一个类别 |
| 动画视频 raw 素材 | 可先导入，再开 [视频转图集](./video-to-atlas) 处理 |

---

## 当心什么

| 当心 | 说明 |
|---|---|
| 类别选错 | 游戏加载规则按目录分，错目录会导致面板里找不到 |
| 同名覆盖 | 覆盖前确认没有场景、角色仍绑旧文件 |
| 未压缩的超大图 | 背景图可先导入，必要时再用图片缩放出游戏尺寸 |
| 只导入不登记 | 文件进了工程，角色/场景字段还要手填或重选 |

## 工作流

```mermaid
flowchart LR
  DL[下载夹 / 协作包] --> ING[分类导入]
  ING --> BR[资源浏览器 核对]
  BR --> REG[主编辑器 登记引用]
  REG --> PV[F5 预览验证]
```

## 雾津例子

1. 画师发来城隍庙夜祭背景 `chenghuang_night.png`，分类选 **环境背景** 导入。
2. 音效师给更鼓「咚」一声，分类选 **音效** 导入。
3. 导入后开资源浏览器，确认两张都在预期目录。
4. 主编辑器 **场景** 里城隍庙场景背景指向新图；**音频** 面板或动作里引用新音效 id。
5. F5 预览夜祭场景，听更鼓是否与画面节奏合拍。
