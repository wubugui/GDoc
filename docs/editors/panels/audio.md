# 音频面板

雾津的雨声、城隍庙的钟声、叫魂长按时按住的那声音——都在**音频面板**登记。读完这页你能：登记一条新音效、把它接到场景或信号 Cue 上、并且清楚知道**这个面板的表单只能编辑音源文件路径，音量和循环这类你没在这儿动过的字段，保存时会原样保留，不会被清掉**。

---

## 这是什么（30 秒看懂）

音频面板就像雾津的一本**声景登记簿**：它不放实际的声音文件（那是资源管线的事），它只负责给每一段声音起个**统一的名字（id）**，指向对应的音源文件，其它面板要用什么声音，都靠这个 id 去认，不靠文件名本身。

它分四类账：**BGM**（背景音乐）、**ambient**（环境循环音，比如雨声、风声）、**sfx**（短音效，比如脚步、翻页声）、**系统音效**（系统按键映射，比如点确认要响哪个音效）。场景选 BGM、临场长按填按住音效、过场和信号 Cue 播音效，全部都是回来这本"登记簿"里查 id。

---

## 入门：手把手做第一次

最常做的事：新登记一个音效，接到一个信号 Cue 上。

```mermaid
flowchart LR
  A[打开主编辑器] --> B[资源 -> 音频]
  B --> C[sfx 表添加条目]
  C --> D[填 id]
  D --> E[文件选择器选源文件]
  E --> F[Apply 保存]
  F --> G[信号 Cue 里选此 id 播放]
  G --> H[预览触发听效果]
```

1. 打开主编辑器：`资源 → 音频`。
2. 切到 sfx 这张表，点**添加**，id 填「sfx_temple_bell」这种一看就懂用途的名字。
3. 用文件选择器挑一个已经进了工程资源管线的音频文件，比如「bell_distant.ogg」。
4. 点 **Apply** 保存。
5. 去[信号 Cue](./cue-signal)面板新建或编辑一条 Cue，比如「temple_bell_ring」，让它的播音效动作选中刚才登记的这个 id。
6. **验证**：运行预览，触发这条 Cue，实际听一下声音是否响起、音量与氛围是否合适。

雾津小例子：城隍庙的钟声应该是远远传来的闷响，不是清脆的近景音效——挑素材时就要照这个方向找，别选了个太清亮的音效毁了氛围。

## 进阶：每一项都讲透

### bgm（背景音乐）

每一条 bgm 有一个**id**（给场景、过场引用用的名字）和一个**源文件**（实际播放的音乐文件）。场景面板选 BGM 时，选的就是这里登记的 id，不是文件名本身——这意味着你可以随时换掉背后的音乐文件而不用动场景配置，只要 id 不变。

### ambient（环境循环音）

登记方式和 bgm 一样，是**循环播放**的一类，专门用来做持续的环境氛围（雨声、市集嘈杂声、风声）。它和 bgm 是两条独立的音轨，场景可以同时挂一条 bgm 和一条或多条 ambient，两者叠加播放，所以调音量、选素材时要考虑叠加后不会互相打架、糊成一团。

### sfx（短音效）

一次性的短促音效，比如脚步声、翻页声、点击声。每条也是**id + 源文件**这个结构。sfx 是被引用最频繁的一类——过场步骤、信号 Cue、临场长按的按住音效，都是从这张表里挑 id。

### 系统音效映射

把系统层面的固定按键（比如 UI 点击确认）映射到某个 sfx 的 id 上。这一项决定了整个游戏系统界面统一的听觉反馈，比如把「确认」这个系统键映射到一声轻木鱼声，能让 UI 交互也带上雾津的气质，而不是通用引擎默认的电子音。

### 关于音量与循环——这里保留你没动过的字段

这个面板的表单里能填的只有每条音频的 id 和音源文件路径，没有摆音量、是否循环这类调节项。保存的时候，编辑器只把音源路径这一项换成你新选的文件，其它没出现在这个面板界面上的字段（音量、是否循环等）原样保留。

也就是说：

- 想调整音量这类设置，走支持渠道（找程序）在这个面板之外处理。
- 素材本身的响度、氛围最好在**制作音频文件的阶段**就调好，省得指望编辑器后期微调。

### id 与文件名的关系

引用永远认的是 **id**，不是背后的文件名。换文件可以随便换（只要新文件音质合适），但 id 一旦被别处引用了，尽量不要轻易改名字，否则会让所有引用它的场景、Cue、过场一起断掉。

### 资源同步

如果文件选择器里找不到想要的音频文件，或者预览没有声音，大概率是这个资源还没有同步到本地（没有 pull 下来）——先确认资源管线同步完成，再回来登记。

### 和别的面板怎么配合

- 和[场景](./scene)配合：场景顶层可以直接选一条 bgm、挂若干条 ambient。
- 和[过场](./cutscene)配合：过场的步骤里可以插入播放音效的动作。
- 和[信号 Cue](./cue-signal)配合：Cue 触发时常常伴随一段表现音，就是靠这里的 sfx id。
- 和[临场长按](./pressure-hold)配合：长按蓄力过程中持续响的"按住音效"，也是从这张表里选。

### 效率与老手技巧

- 给 id 起名坚持"语义清楚、按类型加前缀"的习惯，比如`bgm_`、`amb_`、`sfx_`、`ui_`开头，团队协作时一眼能分清是哪一类。
- 删除某条登记之前，务必先搜一遍场景、过场、Cue、临场长按里有没有还在引用这个 id，避免删完之后某处突然没了声音却找不到原因。
- 不要让同一个用途重复登记两个相近的 id（比如同样的钟声登记了两条），容易在下拉列表里造成选择困惑。
- Apply 之后不要只在编辑器内部点一下预览就算完事，一定要去真正触发这段声音的场景/Cue/过场里实际听一遍，因为叠加播放、音量平衡这些问题只有在真实场景里才暴露得出来。

## 危险区与边界

| 边界 | 说明 |
|---|---|
| **面板表单只编辑源文件** | 音量、是否循环这类没在表单里出现的字段，保存时原样保留 |
| **想调音量这里没有入口** | 面板没提供音量输入框，真要调得走支持渠道（找程序） |
| **id 与文件名要分清** | 所有引用认的是 id，不是文件名，换文件可以，但改 id 名字会让引用处一起断 |
| **资源未同步会显示异常** | 文件选择器里选不到、预览没声音，先确认资源已经拉取到本地 |
| **删除前要搜索引用** | Cue、过场、场景、临场长按都可能还在用这个 id |

更系统的"哪里改了会丢"说明，见[危险区](../concepts/danger-zone)。

## 常见问题

| 现象 | 原因 | 怎么办 |
|---|---|---|
| 游戏里完全没有声音 | 资源没同步，或者引用的 id 写错了 | 先同步资源，再核对各处引用的 id 是否和音频表一致 |
| 想调音量但面板里找不到入口 | 这个面板本身没有调音量的输入框 | 走支持渠道（找程序）在别处调整 |
| 触发 Cue 却没有播音效 | Cue 里填的 sfx id 和音频表对不上 | 回音频表核对准确的 id 拼写 |
| 环境音和背景音乐互相打架 | 两路声音同时满音量叠加 | 到具体场景里实际试听，调整素材本身的响度 |
| 删掉一条 sfx 后系统按键变哑巴 | 系统音效映射还指向被删的旧 id | 改映射指向新 id，或者恢复被删的条目 |
