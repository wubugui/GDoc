# 档案面板

玩家在游戏里翻开的**人物簿**、**见闻录**、**杂书匣**、线装**书籍**，文案全在**档案面板**编。读完这页你能：新建一条带插图的见闻、给人物簿填出解锁分段的印象与已知信息、搭出一本三级结构的书，并且清楚切换条目、切换子 Tab 会不会丢改动。

---

## 这是什么（30 秒看懂）

把整个游戏想成雾津的一座衙门：场景是街巷，图对话是人和人当面说的话，而**档案面板管的是"架阁库"**——衙门后头存档案、录口供、抄志怪的地方。玩家想回顾"关二狗是什么来历""渡口捞尸人有什么闲话""雾津方物略里记了什么古怪"，翻的都是这里编出来的条目。

它和图对话最大的区别：图对话是**当场说的话**，随对话流程走一次就过去了；档案是**留得住、能反复翻**的文字，还能带**解锁条件**（什么时候才让翻）、**首次阅读动作**（第一次读到要顺便发生什么，比如给个旗标）、以及人物专属的**印象与已知信息**（分段展示、带条件）。书籍还多一层——它是**书 → 页 → 条**的三级结构，比见闻录更适合装长篇古卷。

还有一个全站独一份的能力：**插图**。所有富文本框里能插的引用（名字、物品、旗标……）到处都有，唯独"插入一张图"这个按钮，**只在档案编辑器的正文里才有**。想给一段文字配画面，认准档案。

## 入门：手把手做第一次

最常做的事：新建一条**见闻录**，配一张插图，读完给玩家记一个旗标。

```mermaid
flowchart LR
  A[打开主编辑器] --> B[资源 -> 档案]
  B --> C[切到 见闻 Tab]
  C --> D[添加条目]
  D --> E[填 title / content]
  E --> F[正文工具栏插入插图]
  F --> G[设 unlock 条件]
  G --> H[设首次阅读动作]
  H --> I[Apply 保存]
  I --> J[预览里验证]
```

1. 打开主编辑器：`资源 → 档案`。
2. 顶部切到**见闻**这个子 Tab（和人物/文档/书籍是并排的几个 Tab）。
3. 点**添加**，标题填「渡口捞尸人闲话」。
4. 正文（content）里写故事，点富文本工具栏的**插入引用**，选**插图**项，挑一张雾津水彩风格的插图短名插进去——这一步别的面板做不到，只有档案能这样直接插图。
5. 设置**解锁条件**（unlock）：比如要求玩家持有"湿鞋"这个旗标或物品。
6. 设置**首次阅读动作**：加一条"记录阅读进度"的旗标动作，让系统记住玩家读过了。
7. 顺手填一下分类（category）和来源（source），方便玩家侧筛选。
8. 点 **Apply** 保存。
9. **验证**：运行预览，用一个还没满足解锁条件的存档打开见闻录，应该看不到或灰显这条；换一个满足条件的存档，应该能看到、看到插图排版正常、并且旗标被记上了。

雾津小例子：这条「湿鞋来历」见闻只有玩家真的从水里捞到湿鞋之后才该读到——先有实物线索，再有文字解释，顺序对了才有"原来如此"的味道。

## 进阶：每一项都讲透

### 人物簿（characters）

每个人物条目能填：

- **名（name）**：人物显示名，比如「关二狗」。
- **头衔（title）**：副标题，比如「渡口摆渡人」，没有可留空。
- **解锁条件（unlock）**：决定玩家什么时候能在人物簿里看到这一条。走的是全站统一的条件控件，可以填旗标（数值比较）、任务状态（未开始/进行中/已完成）、剧本阶段状态、剧本台词状态、叙事状态机是否到达过某状态，这几类还能用"全部满足/任一满足/取反"互相嵌套，嵌套层数很深，基本够用。
- **首次阅读动作**：玩家第一次打开这条人物简介时顺带发生的事，走的是全站统一的动作控件——可以是记旗标、播一段提示音、弹一段文字，也可以是更复杂的嵌套动作（比如"先判断一个条件分两路各自触发不同效果"）。这里**不写就是白读**：玩家读了，但游戏没留下"他读过"的痕迹，下次条件判断还是会当他没读过。
- **印象（impressions）**：一组"条件 + 一段文字"，随着剧情推进逐条追加，玩家会看到人物印象越写越厚。**每一条印象都有独立的删除按钮**，写错了直接删掉重写就行，不用靠再加一条去覆盖。
- **已知信息**：同样是"条件 + 文字"的分段列表，适合拆成"外貌""脾气""来历"这种小节，每段自己的解锁条件独立控制。

### 见闻（lore）/ 文档（documents）

- **标题（title）**：条目标题。
- **正文（content）**：富文本，**这里能插图**——工具栏的"插入引用"菜单里除了名字、物品、旗标这些标签，还有插图这一项，是全站唯一能这么干的地方。长文配图务必放在这里做，不要指望在图对话或别的面板里插图。
- **来源（source）**：给条目标个出处，比如"庙祝口述""渡口捡到的残页"，方便玩家理解可信度。
- **分类（category）**：给条目分组，方便玩家在见闻录里按类别筛选查找。

### 书籍（books）

书籍是**书（Book）→ 页（Page）→ 条（Entry）**的三级结构，适合装线装古卷这种需要"翻页"体验的长文：

- **Book**：书的顶层，一本书的标题、基本信息。
- **Page**：书里的一页，**有独立的删除页按钮**，用不上的一页可以直接删掉；删之前确认一下这页里的条目确实不需要了。
- **Entry**：页面里的具体条目，可以增删，适合放三条短志怪、三段残句这种颗粒度。

### 通用行为：子 Tab 切换与 Apply

档案面板顶部人物/见闻/文档/书籍这几个 Tab 是分开管理的。切换条目、切换这几个子 Tab 时，你正在编辑的内容会先被带进内存里。**要把改动真正写到磁盘上，最终还是得点 Apply 整体保存**：如果什么都没 Apply 就直接关掉编辑器，没保存的这部分内容会丢。养成"编完一段就点 Apply"的习惯最稳妥。

### 和别的面板怎么配合

- 和[旗标](./flags)配合：解锁条件、首次阅读动作大多落在旗标上，先想清楚要用哪个旗标记录"读过没读过""解锁没解锁"。
- 和[文本库](./strings)配合：档案正文里如果要引用一句全站统一的短语，可以走文本库的字符串引用标签，不用在每条档案里重复打字。
- 和[图对话](./dialogue-graph)分工：当场说的台词走图对话，事后能反复翻的长文字走档案，别把大段说明文字硬塞进对话框。
- 和[文档揭示](./doc-reveal)配合：如果想做"模糊图逐渐清晰"的效果（比如一张残破的地图慢慢显影），那是文档揭示面板的活；档案负责的是文字条目本身。

### 批量与效率窍门

- 见闻/文档条目多的时候，善用**分类**和**来源**两个字段做检索，写十几条以上就会体会到分类的价值。
- 人物簿的印象条目建议先在纸上或文档里想好完整的推进顺序（比如"初见→熟识→知晓身世"三段），一次性按顺序添加，即便顺序想错了也能直接删掉重排，不用太紧张。
- 书籍的页和条目都能增删，放开手脚调整结构，删之前留意一下这页/这条内容是不是真的不需要了。
- 解锁条件和首次阅读动作强烈建议成对检查：新建条目时问自己"这条什么时候能看""看到之后要不要留痕迹"，两个问题都想清楚再 Apply。

## 危险区与边界

| 边界 | 说明 |
|---|---|
| **插图按钮只有档案有** | 别的面板富文本框里想插图，只能手打插图短名（没有可视化引导），正经配图请在档案里做 |
| **印象（impressions）能增能删** | 每条都有独立删除按钮，写错了直接删掉重写，不用靠新增覆盖 |
| **书籍的 Page 能删** | 有删除页按钮，多余的页可以直接删掉 |
| **切换条目、切换子 Tab 不会丢改动** | 有自动提交保护，改动先进内存带着走；但要真正落盘，最终还是要点 Apply 整体保存 |
| **首次阅读动作漏填** | 玩家读了但游戏没留痕迹，之后的条件判断可能仍当作没读过 |

更完整的"哪里改了会丢、哪里编辑器根本够不到"的全局说明，见[危险区](../concepts/danger-zone)。

## 常见问题

| 现象 | 原因 | 怎么办 |
|---|---|---|
| 插图插不进对话框里 | 插图按钮只有档案编辑器有 | 把这段文字改放到档案里做，或退而求其次手打插图短名 |
| 印象条目写错了想删掉 | 这条本身有独立删除按钮 | 直接点删除，重新写一条 |
| 关掉编辑器发现没保存的内容不见了 | 编辑内容还没点 Apply 整体保存 | 切换条目、切换子 Tab 不会丢改动，但退出前要记得点 Apply 才会真正写到磁盘上 |
| 条目在游戏里一直不解锁 | 解锁条件没满足，或条件本身填错 | 用满足/不满足两种存档分别在预览里测试 |
| 玩家读过了但旗标没变 | 首次阅读动作没填或填错 | 回条目补上对应的记旗标动作 |
| 书籍想删掉一页瘦身 | 可以 | 直接点这一页的删除按钮 |
