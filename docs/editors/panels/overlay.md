# 叠图面板

闪白之外，你有时还要在画面上贴一张符咒、一层雾霭、一张旧照片——**叠图**（overlay images）就是给这类"贴图"登记一个好记的短名字，供别处的动作直接喊名字调用，不用每次都重新翻文件夹选图。读完这页你能分清"这本登记册"到底管得到哪几处贴图、管不到哪几处，别再改了半天没反应。

---

## 这是什么（30 秒看懂）

把叠图面板想成庙祝抽屉里编好号的符纸：做法事时喊一声"三号符"，徒弟就把对应那张递上来，不用每次现画现找。**叠图面板**做的就是"给一张图起个短 id，指到工程里的图片路径"这一件事——**只是一张"短id → 路径"的登记表**，本身不产生任何画面效果。

真正让图出现在屏幕上的，是别处的动作或步骤——**呈现叠图**、**叠化叠图**这两个动作可以直接拿这张登记表里的短 id 当"选图"用；但要留心：**不是所有"往屏幕上贴图"的地方都认这张登记表**。过场里"闪一张图"的步骤、[文档揭示](./doc-reveal)的模糊图/清晰图，各自有自己独立的选图框，跟这本登记册不是一回事——这是这块面板最容易搞混的地方，进阶部分会逐条掰开讲清楚。

---

## 入门：手把手做第一次

1. 打开主编辑器（`./dev.sh editor`）→ **资源 → 叠图**。
2. 在列表里点"添加"新增一行：填一个短 id（如 `overlay_mist_light`），点旁边的文件选择按钮，挑一张带透明通道的 PNG（比如一缕雾纹）。
3. **Apply** 保存这本登记表。
4. 去[信号 Cue](./cue-signal) 或图对话的动作编排里加一条**叠化叠图**（或**呈现叠图**）动作：图片框里直接输入你刚登记的短 id `overlay_mist_light`（不用点浏览选文件），系统运行时会自动去这本登记表里查出实际路径。
5. 运行预览，触发这条 Cue，看雾气半透明地叠在画面上，边缘与背景是否自然融合。

```mermaid
flowchart LR
  A[叠图面板：登记短id→路径] --> B[呈现叠图/叠化叠图 动作]
  B --> C[图片框里输入短id或浏览新图]
  C --> D[运行预览触发]
  D --> E[画面上看到叠层]
```

**雾津小例子**：鬼打墙进场想让一个纸人剪影一闪而过——登记短 id `overlay_paper_man` 指向剪影 PNG，在鬼打墙进场的信号 Cue 里加一条**呈现叠图**动作，图片框填 `overlay_paper_man`，透明度、位置在动作参数里调；玩家一进鬼打墙就会看到剪影闪现。

## 进阶：每一项都讲透

### 本面板的两个字段

- **短 id**：策划自己起的代号，字母数字下划线即可，如 `overlay_fog`、`overlay_talisman`、`overlay_old_photo`。id 一旦被别处动作引用就尽量别改名——改名不会帮你同步别处已经填好的旧 id 字符串。
- **路径**：指向工程内的图片文件。换图只要保持 id 不变，所有引用它的地方会自动换成新图，不用逐条改。

这本表就是简单的表格式增删改，没有更复杂的结构。

### 叠图登记表到底管得到哪些地方（讲透三种"贴图"机制）

这是本页最关键的一节，务必读完再动手，否则很容易"改了登记表但画面没变"。

1. **呈现叠图动作**：参数里有一个 **id**（这是"操作把手"，跟"关闭叠图"共用同一个 id，用来指定"关哪一层"，**不是图片本身**）、一个 **图片** 框（这里才是真正决定贴什么图的地方——可以直接输入登记表里的短 id，也可以点浏览选一张新图，两种都行；运行时只要看到的内容不是以 `/` 开头的完整路径，就会去登记表里查这个短 id 对应的路径）、**屏幕位置**（x/y 百分比，决定图片中心贴在屏幕的哪个百分比位置）、**宽度百分比**（图占屏幕宽度的百分之几，高度按原图比例自动算，不用你换算）。
2. **叠化叠图动作**：**id** 同上（操作把手，跟"关闭叠图"共用）；**起始图**、**目标图** 两张图（同样可以填登记表短 id 或浏览新图）；**叠化时长**（毫秒，图从起始渐变到目标要多久）；**起播延迟**（毫秒，动作触发后先等多久再开始渐变，0 就是立即开始）；位置/宽度百分比同上。这个动作用来做"一张图慢慢变成另一张图"的效果，比如信件手写慢慢变清晰。**但项目现在建议**：这类"模糊变清晰"的效果优先用 [文档揭示](./doc-reveal) 面板去做（它自带条件判断和存档记录），不要再在动作里手写两张图路径来对付——文档揭示是为这个场景专门设计的。
3. **关闭叠图动作**：只填一个 id，把之前那层"呈现"或"叠化"打开的图层关掉。
4. **过场里"闪一张图"的步骤**：这是[过场](./cutscene)自己的独立步骤，有自己单独的选图框，**直接选文件，不查这张登记表**——哪怕你在这里登记了同名短 id，过场步骤那边照样得自己重新选一次图片文件。两者不共用素材登记。
5. **文档揭示的模糊图/清晰图**：同样是各自独立选文件，**不查这张登记表**。文档揭示里那个内部代号字段，是给揭示动画本身起的一个内部代号（默认按"揭示_对应条目 id"的格式自动生成），跟这本登记表完全没关系，绝大多数情况都不用填。详见 [文档揭示](./doc-reveal) 页。

一句话记忆法：**叠图登记表只服务"呈现叠图 / 叠化叠图 / 关闭叠图"这三个动作的图片框**，过场闪图和文档揭示的图各管各的。

### 什么时候该登记、什么时候不必

- 同一张图会在多个过场/信号 Cue 里反复用到（比如"雾"这张图在好几段演出里都要闪一下）→ 值得登记短 id，以后想统一换风格，改一处路径大家都跟着变。
- 只在一个地方用一次的图 → 直接在那个动作里浏览选文件即可，不用特地跑来登记。
- 想做"模糊到清晰"的信件/告示效果 → 别用叠化叠图手写两张路径，去用 [文档揭示](./doc-reveal)。

### 和别的面板/动作怎么配合

| 面板/动作 | 关系 |
|---|---|
| 呈现叠图 / 叠化叠图 / 关闭叠图（动作） | 图片框认这张登记表的短 id |
| [过场](./cutscene) 闪一张图 | 自己独立选图，不认这张表 |
| [文档揭示](./doc-reveal) | 模糊图/清晰图自己独立选图，不认这张表；其内部代号字段也不是这张表的引用 |
| [信号 Cue](./cue-signal) | 常见的挂载呈现叠图/叠化叠图动作之处 |
| [滤镜](./filters) | 滤镜改的是整体画面色调，叠图改的是叠加的具体内容，两者分工不同 |

### 批量做法与效率窍门

- 项目初期就把常用的一批雾、符、旧照、剪影等图片登记好短 id，命名带上场景/用途前缀（如 `overlay_fog_dock`），下拉选的时候一眼能认出来。
- 大图务必走压缩管线再登记，避免叠图加载慢导致演出卡顿。
- 带透明通道（alpha）的 PNG 优先，避免白底硬边穿帮。

## 危险区与边界

- **这张表本身不是重建区也不是盲区**——它只是一个简单的短 id→路径字典，Apply 保存不会抹掉你没碰过的行。
- 但**它管不到**过场闪图、文档揭示的图片选择——改了本表这两处不会跟着变，这是最容易误解的边界，别把力气使错地方。
- 删除某个短 id 前，编辑器**不会自动帮你找出**哪些"呈现叠图/叠化叠图"动作还在用这个字符串——你得自己去过场、信号 Cue、图对话里逐条查，删漏了会造成某段演出突然黑块或报错。
- 全屏图要留出 UI 安全边，别把对话框、任务栏挡死。
- 更多编辑器整体的可编辑边界见 [危险区](../concepts/danger-zone)。

## 常见问题

| 现象 | 原因 | 怎么办 |
|---|---|---|
| 登记了短 id，过场里闪图还是老样子 | 过场闪图有自己独立的选图框，不查叠图登记表 | 去过场步骤里重新选一次文件 |
| 文档揭示的清晰图改不动 | 文档揭示的图各自独立选文件，不认叠图登记表 | 去[文档揭示](./doc-reveal)面板里改对应字段 |
| 引用短 id 后画面没有叠层 | id 拼错、或图片框填的不是登记表里的 id | 核对拼写，或改成浏览直接选文件 |
| 叠图加载慢或卡顿 | 原图过大未压缩 | 走资源压缩管线后再换路径 |
| 全屏图挡住对话按钮 | 没留 UI 安全边 | 裁切图片或缩小有效显示区域 |
| 删 id 后某段演出报错/黑块 | 还有动作在引用这个旧 id 字符串 | 先改掉引用，或把 id 恢复 |
