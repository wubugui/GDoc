# 动画浏览面板

关二狗走路、庙祝作揖、纸人飘动——动画来自生产管线（视频转图集等），不是在手写 JSON。**动画浏览**让你在主编辑器里 **查** 已产出的动画包：有哪些状态、哪段 clip、预览播一遍，给 [角色登记](./character) 和 [场景](./scene) NPC 选型用。

场景里 anim 表 **只读**；要新动画走资产工作台，不在此面板编。

---

## 这块面板管什么

- **动画包列表**：按角色或文件名浏览。
- **状态/片段预览**：点选播放（视实现）。
- **只读**：不改曲线、不导出、不删文件。

---

## 怎么打开

1. `./dev.sh editor` → **资源 → 动画浏览**。
2. 选动画包 → 选状态预览。
3. 记下包名/id 回角色登记填。

## 工作流

```mermaid
flowchart LR
  PROD[视频转图集等] --> PKG[动画包]
  PKG --> BR[动画浏览 查]
  BR --> REG[角色登记]
  REG --> SC[场景 NPC]
```

---

## 怎么用

1. 新立绘动画产出后，先开动画浏览确认 **状态名** 与 [场景](./scene) 里 `initialAnimState`、巡逻 `moveAnimState` 一致。
2. 发现缺「作揖」状态 → 回生产补导出，别在场景硬填不存在的状态名。
3. 对比两套动画包选更贴雾津气质的再登记。

---

## 怎么改 / 删

- **本面板不能改删**；替换资源走资产管线重新导出。

---

## 当心什么

| 当心 | 说明 |
|---|---|
| 状态名大小写 | 场景填错播默认或卡第一帧 |
| 只浏览不登记 | 场景 NPC 下拉仍空 |
| 与碰撞无关 | 动画长短不自动改交互范围 |
| 绕过面板手写动画表 | 场景里只读引用，下次资产导出可能被覆盖 |

---

## 雾津例子

1. 浏览 `guan_ergou_anim` 确认有 `idle_dock`、`walk`。
2. [角色登记](./character) 关二狗绑此包。
3. 场景码头 NPC initialAnimState `idle_dock`；巡逻 moveAnimState `walk`。
4. 纸人 NPC 用另一包，浏览里确认 `float` 循环是否顺滑。

## 实操检查清单

- [ ] 新动画包入库后，先在本面板逐个点选预览各状态
- [ ] 核对站立、行走、剧情专用三类常用状态是否齐全
- [ ] 确认循环动作首尾衔接自然，无明显跳帧或闪一下
- [ ] 把各状态名的拼写准确记下，与角色登记、场景 NPC 填写完全一致
- [ ] 对比两套候选包，选更符合雾津阴湿气质的那套再登记
- [ ] 非人角色（纸人、游魂）的飘浮、作揖类动作时长是否合理
- [ ] 发现缺状态时已走资产管线补导，而非在场景硬填不存在的状态
- [ ] 浏览完成后回到角色登记完成绑包，避免只看不登记
- [ ] 巡逻用与初始站立用的状态分别记清，勿混填
- [ ] 确认本面板确为只读浏览，误改需求应转资产工作台
- [ ] 预览时留意动画锚点与场景碰撞范围无关，勿指望动画长短改交互

## 常见问题

| 现象 | 原因 | 怎么办 |
|---|---|---|
| NPC 站立卡在第一帧 | 场景填的状态名在包里不存在 | 回动画浏览核对正确名称，改场景或补导出 |
| 预览能播、游戏里不动 | 角色登记未绑此动画包 | 先在角色登记选包并保存，再查场景 NPC 身份 |
| 行走像在滑步 | 步频与场景移动速度不匹配 | 与场景策划对齐速度，必要时换状态或调速 |
| 列表有包但预览黑屏 | 本地资源未同步 | 拉取最新资产后重开面板 |
| 两套包状态名对不上 | 生产流程命名不统一 | 定一套标准，另一套对齐或废弃 |

---

## 预览验证

1. 打开动画浏览，选中目标包，逐个点状态播放一遍。
2. 记下 idle、walk 及剧情专用状态的准确名称。
3. 到角色登记绑定同一包并保存。
4. 在对应场景放 NPC，初始与移动状态填刚才记下的名称。
5. 运行预览，观察站立循环与巡逻行走是否顺畅。
6. 若项目有昼夜或位面切换，再确认动画未被错误覆盖。

---

在雾津寻狗记里，关二狗靠渡口缆桩待机时，你应能在浏览里找到对应的靠桩站立循环；若无此状态，码头氛围会弱一大截。纸人的飘浮状态建议连播三遍看是否无缝，否则鬼打墙巷口会显假。庙祝作揖若只有前半段，遭遇里作揖选项的反馈可能对不上——浏览阶段就要发现并补全。货郎挑担动画若与关二狗共用包，预览里要确认不会串脸。
