# 用运行预览验证改动

改了对白、摆了场景、调了滤镜——若不进游戏看一眼，心里总悬着。**运行预览**就是编辑器里的「当场验货」：保存 → 起游戏 → 走到触发点 → 确认。这一页把三种预览方式和常见习惯一次说清。

---

## 读完你能做到什么

- 用 **F5** 在主编辑器里内嵌预览
- 用 **Ctrl+F5** 独立窗口、或浏览器里已开的游戏验证
- 知道「先保存再预览」和「热重载」的差别
- 养成改完必验的闭环，少踩「改了没生效」

---

## 三种预览方式

| 方式 | 怎么开 | 适合什么时候 |
|---|---|---|
| **内嵌预览** | 主编辑器按 **F5** | 最常用；改完立刻在同窗口看 |
| **独立窗口** | **Ctrl+F5** | 想双屏：一边编辑一边全屏玩 |
| **浏览器游戏** | `./dev.sh game start` 后切浏览器 | 已长期开着游戏、靠自动刷新 |

停止内嵌或独立预览：**Shift+F5**。

```mermaid

  A[改完内容 · Ctrl+S 保存] --> B{怎么验?}

  C --> F[走到触发点确认]

```

---

## 第 1 步：打开主编辑器

```bash
./dev.sh editor
```

左侧导航树末尾有 **Game（运行预览）** 页——F5 后游戏画面嵌在这里。

---

## 第 2 步：内嵌预览（F5）

1. 改完当前面板，**Ctrl+S** 或 **Ctrl+Shift+S** 保存
2. 按 **F5**
3. 编辑器会先保存，再启动游戏开发服，把画面嵌进 Game 页
4. 用键盘操作走进场景，触发你对白/热区/商店/小游戏

:::tip[F5 不是万能刷新]
F5 会**重新起预览**。若你已在预览里，小改动有时保存后预览会自动跟上；若完全没反应，**Shift+F5** 停掉再 **F5** 重来。
:::

## 第 3 步：独立窗口（Ctrl+F5）

和 F5 一样先保存再起服，但游戏在**单独窗口**打开——适合：

- 外接显示器全屏玩
- 内嵌区域太小，看不清立绘或小游戏

---

## 第 4 步：浏览器 + 热重载

若你早已另开终端跑着：

```bash
./dev.sh game start
```

浏览器里地址一般是 `http://localhost:5173`（端口占用时会变，看终端提示）。主编辑器保存后，浏览器**多数情况**会自动刷新进新数据。

适合长时间开着游戏、频繁改小文案。若改了结构型内容（新场景、新任务链），仍建议 **F5** 完整预览一次。

---

## 验什么、怎么验

按你刚改的内容列检查单：

| 你改了 | 预览时做 |
|---|---|
| 对白 | 走到触发点，看台词、选项、分支 |
| 场景 / NPC | 走进场景，看位置、立绘、碰撞 |
| 物品 / 商店 | 买一件、用一件、看背包 |
| 滤镜 / 深度 | 看色调、绕障碍物走一圈 |
| 过场 / 视差 | 从触发点播完整段 |
| 小游戏 | 玩通胜利和失败两条路 |
| 档案 / 音频 | 开书匣、听 BGM 与音效 |

改一处，验一处——别攒十处改动最后一起试，出了问题不知道是哪一刀。

---

## 和校验命令配合

预览是「眼睛看」；数据有没有硬错误，可以另跑：

```bash
./dev.sh validate-data
```

校验不过，预览里也可能怪——先修红灯再 F5。详见 [出问题怎么办](./troubleshooting)。

## 进阶：老手的验货习惯

### 别从头玩一遍，直接跳过去

改的是城隍庙深处一句台词，没必要每次都从游戏开头走过去触发。**Web 控制台**（`./dev.sh console`）里带有**场景快捷入口**和**叙事 warp 入口**——前者直接跳进某个场景，后者跳到某个叙事位置，都是从工程数据里读出来的开发用捷径。已经起了游戏的情况下，这比每次重走一遍剧情快得多。

### 配合面板前进后退

预览发现问题、回去改，再验一次——这个来回本身有个顺手的技巧：改完 `Alt+←` 能带你回到刚才那个面板（比如从图对话回到场景），不用在导航树里重新找。改完接着 `F5`，来回节奏比每次手动点导航树快不少。

### 分场景/分状态验证

同一段内容在不同条件下表现可能不一样——比如：

| 变量 | 要验的东西 |
|---|---|
| 不同 [位面](../reference/glossary) | 白天/夜晚气氛组、位面限定的热区是否只在该位面出现 |
| 不同旗标/任务进度 | 条件门控的内容是否该出现的出现、不该出现的不出现 |
| 不同存档 | 新档和老档（已解锁部分内容）表现是否都对 |

只测一条路径容易漏掉分支——尤其是条件、位面相关的内容，至少切两种状态各测一次。

### 先校验、后预览

`./dev.sh validate-data` 跑起来比进游戏走一遍快得多。改动量大时，习惯先跑一次校验清掉硬错误，再进预览看体验细节，比直接进预览排查报错效率高。

### 独立窗口的额外用处

除了双屏，**独立窗口**（`Ctrl+F5`）在场景复杂、内嵌区域画面小看不清（比如立绘细节、小游戏画布）时也更合适；编辑器负担也会轻一些。

---

## 危险区与边界

- **预览不等于数据校验**：看着运行正常，不代表 `./dev.sh validate-data` 一定会过；反过来校验通过也不代表玩法体验没问题——两者要分别跑。
- **结构性改动**（新增场景、新任务链、新建小游戏关卡）有时热重载跟不上，建议 **Shift+F5** 完整停掉再 **F5** 重开，别指望增量刷新。
- 预览里的操作**不会写回编辑器数据**——你在预览里瞎点、走错路，不会弄坏你编的内容；但预览进度也不能当正式存档使用，别指望它帮你"存档"。
- 内嵌预览和独立窗口/浏览器跑的是同一份游戏数据，但**互相之间不共享运行状态**——独立窗口开着不代表内嵌预览也在跑，反之亦然。

## 雾津小例子

你刚改了关二狗在**满堂茶客**的一句台词，又调了茶馆滤镜：

1. 图对话保存 → **F5**
2. 走进茶馆触发对话，对台词
3. **Shift+F5** 停预览，场景面板确认滤镜 → 再 **F5** 只看色调
4. 都对了，这条折子才算落笔

## 常见问题

| 现象 | 原因 | 怎么办 |
|---|---|---|
| 按 F5 没反应 | 未选中「运行与预览 → Game」，或环境未就绪 | 确认导航树选对页；提示环境缺失时先跑 `./bootstrap.sh` |
| 改了内容，预览里还是旧的 | 对应面板没保存，或预览没重启 | 确认 `Ctrl+S`/`Ctrl+Shift+S` 已存；`Shift+F5` 再 `F5` |
| 新场景/新任务预览里没反应 | 结构性改动热重载跟不上 | 完整停掉预览重开，别只指望增量刷新 |
| 内嵌预览操作很卡 | 复杂场景占用编辑器资源 | 改用 `Ctrl+F5` 独立窗口，减轻内嵌负担 |
| 想直接测某段剧情，不想从头玩 | 不知道有跳转入口 | 用 Web 控制台的场景快捷入口 / 叙事 warp 入口直接跳过去 |
| 校验通过但预览里还是不对 | 校验只查数据硬错误，不查体验细节 | 数据没错不代表逻辑对，仍需按检查单走一遍预览 |
