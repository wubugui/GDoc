# 做一个视差过场

有些戏不宜让玩家走着看——进城隍庙前，远山、牌楼、人影宜分层滑过，像卷轴徐徐展开。**视差过场**干的就是这个：多层图各走各的速度，镜头推移时远近错落。这一页从摆图层到挂进过场，走通一遍。

---

## 读完你能做到什么

- 用**视差编辑器**建多层场景、设关键帧与运动轨迹
- 保存后供**过场**调用
- 预览里确认图层速度、时长、衔接
- 在游戏里播一遍完整过场

---

## 视差是什么

> **[视差](../reference/glossary)**：近处动得快、远处动得慢，眼睛觉得有纵深。过场里用多层静态或动态图，按不同速率平移，营造「镜头在扫街」的效果。

视差数据和普通场景是分开的——在**视差编辑器**里编，在**过场**面板里用「播放视差场景」这类步骤触发。

---

## 第 1 步：打开视差编辑器

```bash
./dev.sh parallax-editor
```

浏览器会打开编辑界面（本地端口由工具提示）。渲染和游戏一致，调完所见即所得。

也可 `./dev.sh editor` → **工具 → 外部工具** → **视差场景编辑器**。

---

## 第 2 步：建场景、摆图层

1. **新建或打开**一个视差场景——例如 `城隍庙 entrada`（进庙前外景）
2. **加图层**（从下到上）：
   - 最远：雾津远山、天色
   - 中间：牌楼、树影
   - 近处：石阶、灯笼、行人剪影
3. 每层设**视差速率**——远景 0.2、近景 1.0 是常见起点，按感觉微调
4. 素材先用 [导入一张素材](./import-art) 进库，编辑器里按短名选图

## 第 3 步：关键帧与轨迹

1. 在时间轴上打**关键帧**：镜头起止位置、时长
2. 拖**镜头路径**——横向扫街、缓慢推进、停在庙门前均可
3. 逐层检查：有没有穿帮、边缘是否露白（图不够宽要加长或循环）
4. **保存**——结果写入工程的视差场景库

---

## 第 4 步：挂进过场

```bash
./dev.sh editor
```

**叙事编排 → 过场**：

1. 新建或打开一段过场——如进庙前的引子
2. 在步骤序列里加**播放视差场景**（名称以面板里为准）
3. 选中刚编的视差场景 id
4. 后面可接旁白、切场景、黑场等步骤

过场面板里也有按钮可直达视差编辑器改同一景。

---

## 第 5 步：运行预览

**F5** 触发这段过场：

- 各层速度是否自然
- 总时长和旁白、音乐是否合拍
- 播完是否顺利切到下一个场景或步骤

## 雾津小例子

李天狗带关二狗**头一回见城隍庙**，要一层层揭开牌楼：

1. 视差编辑器建 `chenghuang_approach`，四层：雾、牌楼、街人、石阶灯笼
2. 镜头从远推近，四秒停庙门匾下
3. 过场 `intro_chenghuang` 第一步播该视差，第二步接李天狗旁白
4. **F5** 从新档或调试入口播过场，听旁白是否踩准画面停住那一刻

卷轴拉开，庙才显灵。
