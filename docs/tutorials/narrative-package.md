# 配一个叙事包

这一页带你做一件很实用的事：配一个叙事包，并让玩家进入场景时能接上它。你会用到主编辑器里的三个地方：章节导演、叙事状态机、场景。章节导演负责决定哪一包剧情在某个场景里活跃；叙事状态机负责故事阶段怎么流转；场景进入时，也就是 onEnter，负责玩家一进场时先敲一下门，把当前环境和叙事包接起来。

教程会用雾津茶馆开场做例子。玩家进茶馆，先听到说书人的开场，完成后故事推进，后面背尸线包或码头洋人线包才能按进度出现。你不需要写任何代码，只需要在主编辑器里按顺序填面板、保存、预览。

---

## 这是什么

叙事包适合装「一段在某个阶段才应该出现的剧情内容」。它可以接一张图对话，也可以代表一组已经整理好的叙事内容。关键不在于包里说了多少句台词，而在于它什么时候 live，什么时候 dormant。

章节导演像排班表：它告诉茶馆、义庄、码头这些场景，此刻应该让哪一包候场。叙事状态机像进度尺：它告诉系统故事已经从「初到雾津」推进到「听完茶馆开场」或「码头疑云」。场景进入时动作像门铃：玩家一进茶馆，它先根据当前场景和状态触发你安排好的检查、提示或信号。

三者分工要清楚：

- 章节导演决定哪一包活。
- 叙事状态机决定故事阶段如何推进。
- 场景负责玩家进入时发生什么。
- 图对话负责包内具体怎么说、怎么分支。

如果你把这几件事混在一起，早期可能看着省事，后面就会出现茶馆包一直不退、码头包提前出现、背尸线接不上义庄的情况。

## 入门：手把手做一次

### 第一步：打开主编辑器

在项目根目录启动主编辑器：

```bash
./dev.sh editor
```

启动后先不要开预览。我们要先把叙事包、状态和场景入口接好，再用 F5 做编辑器预览。

### 第二步：准备一个茶馆开场包

进入 **叙事编排 → 章节导演**，新建一条叙事包记录。给它一个你半年后还能看懂的名字，例如「茶馆开场包」。绑定场景选茶馆，绑定内容选你已经准备好的茶馆开场图对话或叙事包。

激活条件先设得简单：故事处于初到雾津，且茶馆开场还没完成。完成条件也要同时准备好：茶馆开场图走完，或开场结束标记已经写下。这样它只在玩家第一次需要听开场时 live，结束后就 dormant，不会每次进茶馆都重播同一段。

填完后点 Apply，再保存。这里的 Apply 很重要，它会把当前记录写回叙事包列表；保存则让整个项目记住这次更改。

### 第三步：在叙事状态机里准备阶段

进入 **叙事编排 → 叙事状态机**。你需要至少有两个阶段：一个代表初到雾津，一个代表茶馆开场已完成。名字不必复杂，但要表达清楚。

然后从初到雾津拉一条迁移到茶馆开场已完成。触发方式可以来自茶馆开场包结束时发出的信号，条件可以保持简单：只要这段开场确实完成，就允许推进。

这里要记住：状态机不替你播放茶馆对白，它只负责「故事阶段变了」。茶馆包里的图对话走完后，把完成信息交给状态机，状态机再推进到下一阶段。后面背尸线包、码头洋人线包、阎王岭包都可以读取这个阶段，决定自己是否该出现。

### 第四步：回到章节导演核对条件

状态机阶段准备好后，回章节导演检查茶馆开场包的激活条件和完成条件。

激活条件应当读「初到雾津」这类早期阶段，并确认开场还没完成。完成条件应当在图对话结束或阶段推进后成立。你要避免两种错误：一是条件太宽，导致茶馆开场包一直 live；二是条件太窄，导致玩家第一次进茶馆时也看不到它。

如果你已经准备背尸线包，可以顺手建第二条记录：绑定义庄或雾津街头，激活条件放在茶馆开场完成之后。先不要把它做复杂，只让它在茶馆开场后能出现即可。这样你能马上验证「一个包退场，另一个包接上」的效果。

### 第五步：在场景里接 onEnter

进入 **物理世界 → 场景**，选择茶馆场景。找到场景进入时，也就是 onEnter 的动作区域。这里不写剧情台词，只做接线：玩家进入茶馆时，先让场景执行进入时动作，再由章节导演判断当前 live 的包，最后让对应的图对话或入口可用。

你可以安排一个轻量的进入提示，例如茶馆环境声、说书声起，或者检查当前是否有 live 的茶馆包。如果当前 live 的是茶馆开场包，玩家进入后就能看到对应入口或直接触发开场；如果包已经 dormant，场景就正常进入后续状态，不再重播开场。

场景进入时动作不要承担太多剧情判断。它像开门时的第一下动作，只负责把场景和叙事包接起来。真正判断哪包活，交给章节导演；真正推进故事阶段，交给叙事状态机；真正说话，交给图对话。

### 第六步：让图对话收尾

打开茶馆开场对应的图对话，确认最后有一个清晰收尾：玩家听完说书、做完选择或看完提示后，写下开场完成的标记，并发出推动叙事状态机的信号。

这里的收尾非常关键。没有收尾，章节导演的完成条件不会满足，茶馆开场包会一直 live；没有推动状态机，后续背尸线包、码头洋人线包可能永远等不到进度。

你不需要把所有后续内容都接在这张图后面。茶馆开场只负责把第一段结束干净，后面由状态机和章节导演自然接出下一包。

### 第七步：保存并用 F5 预览

确认章节导演、叙事状态机、场景和图对话都保存后，按 F5 进入编辑器预览。从新进度开始走：进入茶馆，触发茶馆开场包，完成开场，再退出茶馆重新进入。

你要检查四件事：

- 第一次进茶馆时，茶馆开场包确实 live。
- 开场结束后，叙事状态机确实推进到下一阶段。
- 再次进入茶馆时，茶馆开场包已经 dormant，不再重复开场。
- 如果你准备了背尸线包或码头洋人线包，它们是否在茶馆开场完成后按条件出现。

如果四件事都对，这条叙事包就接上了。

## 进阶：把茶馆、背尸、码头串起来

### 茶馆开场包

茶馆开场包只做入门引导：玩家第一次进茶馆，听说书人讲一段，得到雾津的第一层气味。它的完成条件要明确，最好在图对话自然收尾时成立。不要让它依赖玩家离开茶馆才完成，否则玩家在茶馆里回头点同一个入口时，可能又看到开场。

### 背尸线包

背尸线包适合在茶馆开场后激活。你可以让它绑定雾津街头或义庄：街头负责接到活计，义庄负责交差。若它跨两个场景，建议拆成两条记录，各自绑定自己的场景。这样街头只管接活，义庄只管交付，不会在一个包里塞太多场景逻辑。

背尸线完成后，可以归档活计次数，也可以推动某个支线阶段。注意活计本身可以重复，但当前激活的一单要清楚，别让多个背尸目标同时争夺任务提示。

### 码头洋人线包

码头洋人线适合在背尸或茶馆后续信息出现之后再激活。它可以绑定码头，让玩家到了码头才看到相关入口。激活条件不要只写「码头已解锁」，还要确认故事进度已经允许洋人线露头，否则玩家可能刚到码头就撞见太后面的信息。

码头包结束时，不一定立刻把玩家推向阎王岭。你可以先让状态机进入「码头疑云」一类阶段，再由章节导演决定城隍庙、义庄或街头是否出现新的包。

### 阎王岭包

阎王岭包适合放在更后面。它的激活条件应该比普通地点更严格：既要地点开放，也要故事阶段、任务或线索到位。它一旦提前 live，玩家会过早知道山口相关信息，破坏雾津的压迫感。

你可以先只做山口入口包：玩家第一次来到阎王岭山口时触发短引导，完成后休眠。后续更深的内容再拆成新的包，不要把整条岭上剧情都塞进一个包。

## 危险区与边界

- 不要把叙事包当成任务。任务告诉玩家要做什么，叙事包告诉场景哪段内容活跃。
- 不要把状态机阶段塞进图对话里各自判断。包内可以读阶段，但阶段推进应当统一交给叙事状态机。
- 不要让场景 onEnter 承担整段剧情。它只负责接线和进入时动作。
- 不要忘记完成条件。没有完成条件的茶馆开场包会反复出现。
- 不要让后续包只靠地图开放来激活。地图能去，不代表剧情该露。
- 不要只预览第一次进入。一定要完成后再进一次，确认包已经退场。

边界越清楚，越容易扩。茶馆、背尸、码头、阎王岭都可以按这个模式往下接：先定故事阶段，再定场景里的 live 包，再让包内内容收尾并推动下一阶段。

## 常见问题

### F5 预览里第一次进茶馆没触发？

先检查茶馆开场包是否绑定茶馆，再检查激活条件是否在新进度下满足。然后看场景进入时是否真的接到了章节导演判断，最后确认 Apply 和保存都做过。

### 开场播完后，第二次进茶馆还在播？

完成条件没有成立，或图对话收尾没有写下完成信息。回到图对话检查结尾，再回章节导演检查完成条件是否能读到这个结果。

### 状态机没有推进？

多半是茶馆开场结束时没有发出推动信号，或迁移等待的信号和图对话发出的信号不是同一个。用状态机调试查看当前阶段，确认它是否还停在初到雾津。

### 背尸线包提前出现了？

激活条件太宽。给它加上茶馆开场完成、相关任务已接或对应线索已出现这类门槛。不要只靠场景存在来判断。

### 码头洋人线在地图开放后立刻露头，怎么办？

把地图开放和剧情露头分开。地图开放只说明地点可去；洋人线包还应读取故事阶段或线索条件，确认玩家已经有理由注意这条线。

### 我能不能一个大包管完整章节？

可以，但不推荐。雾津这种场景和支线交错的结构，更适合按场景和阶段拆小包。茶馆开场、背尸交付、码头疑云、阎王岭入口分别成包，维护起来更清楚。
