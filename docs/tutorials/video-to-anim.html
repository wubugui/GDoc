把视频做成角色动画

雾津街上的人物不能只会站桩——走路、挥手、蹲点查案，都得有动画。若你手里是一段拍好的或生成的角色视频，这一页教你变成游戏里能播的精灵动画：抽帧 → 拼图集 → 登记状态 → 预览 → 进场景验证。

这是什么（30 秒看懂）​

把这件事想成把一段皮影戏录像，剪成一张一张定格画片，再钉成一叠能快速翻动的「西洋镜」画册。视频转图集这个工具帮你从视频里挑出关键的几帧（比如「走路」的一个完整循环需要哪几张画面）、拼成一张大图（图集），再写一份说明书告诉游戏「按什么顺序、多快翻这些画片」。游戏运行时不会真的播放视频，而是按这份说明书快速切换图集里的一格格画面，看起来就像人物在动。

这条管线的核心价值是手动控帧：你能精确决定每一帧长什么样、删掉哪些糊的或多余的帧，比全自动生成的动画更可控，适合关键角色、动作分支多的场合。

读完这页你能：

  • 用视频转图集从视频里抽出动作帧。
  • 导出图集与动画描述，供游戏加载。
  • 在主编辑器动画浏览里核对各状态（站立、行走等）。
  • 用动画预览和运行预览确认角色动起来，脚点不飘、循环不跳变。

入门：手把手做第一次​

整条管线长什么样​

图集：把很多小帧拼成一张大图，游戏按格切出来播——省资源、加载快。

第 1 步：素材先入库​

视频原始文件建议先按 导入一张素材 的流程分类导入进工程，方便管理和追溯来源。

第 2 步：打开视频转图集​

这个工具从主编辑器菜单进（没有单独的 ./dev.sh 命令）：

./dev.sh editor

菜单栏 → 工具 → 外部工具（新进程） → 选 视频转图集。

也可以 ./dev.sh console，在工具列表里找同名入口。

第 3 步：建工程、抽帧​

  1. 新建或打开一个工作区（工具会记住上次进度，工作区数据独立存在一个目录里）。
  2. 导入视频——例如关二狗在码头蹲点张望的片段；一个工作区里可以并存多段视频，各管各的动作。
  3. 在时间轴上标出要用的区间（哪几秒算「走路」，哪几秒算「站立」）。
  4. 预览抽出来的帧，删掉糊的、重复的、动作跳变的帧——工具支持逐帧微调，帧内容不满意可以直接在这一步剔除。
  5. 为每个动作起名：常见有 idle（站）、walk（走）、run（跑），命名要和后面场景里要填的状态名保持一致，提前想好命名规范能省很多回头改的功夫。

多段视频可以并进同一个角色工程，各管各的状态；同一个角色不同动作用不同视频拍摄也完全没问题，只要落到同一个工作区。

操作示意​

第 4 步：导出​

点导出，工具会生成：

  • 一张图集大图（所有帧格排在一起）。
  • 一份动画描述（哪几格、多快、循不循环）。

导出位置按角色/动画包归好类，方便后面登记。主编辑器里动画浏览面板的图集字段是只读的——要换图集，回这个工具重新导。

第 5 步：动画浏览里核对​

./dev.sh editor

左侧 → 资源与本地化 → 动画浏览：

  1. 找到刚导出的动画包。
  2. 看各状态的帧序、帧率、是否循环。
  3. 世界尺寸不对可在面板里微调（图集本身仍须回视频转图集改）。

想单独、快速地看某个角色某状态，用动画预览（渲染规则与游戏完全一致，能看清脚点、循环衔接、滑步等动画浏览小窗看不清的问题）：

./dev.sh anim-preview

浏览器里选角色和状态，所见与游戏里一致。

第 6 步：挂到角色、进场景​

  1. 角色登记面板 → 选中角色（如关二狗）。
  2. 把动画包指到刚导出的那份。
  3. 场景面板 → 场景里的 NPC 选同一角色身份，初始状态、巡逻状态填与视频转图集里一致的状态名。
  4. F5 运行预览，看走路、站立是否顺滑、脚底是否贴地。

脚底飘、比例怪？多半是抽帧区间或世界尺寸要再调一轮。

雾津完整实例：糖画师傅的招手动画​

要给城隍庙前卖糖画的师傅补一套「招手揽客」动画：

  1. 视频原始素材先按 导入一张素材 分类导入。
  2. 外部工具打开视频转图集，导入招手视频，标区间，状态名起 wave。
  3. 预览抽出的帧，删掉手部动作模糊的那几帧。
  4. 导出后进动画浏览，确认 wave 循环自然，首尾衔接不跳变。
  5. 角色登记里师傅的动画包加上 wave。
  6. 场景里师傅 NPC 走近能对话；若还要自动招手，在区域或 NPC 行为里编排播放该状态。
  7. ./dev.sh anim-preview 先看大图细验，再 F5 进城隍庙验证实际效果。

人动起来，街才活。

进阶：每一项都讲透​

工作区与帧管理的进阶用法​

  • 多段视频并入同一工程：一个角色的不同动作（走路、站立、特殊动作）完全可以分别拍摄、分别导入，都放进同一个工作区管理，导出时统一打成一个动画包。
  • 首尾帧对照预览：抽帧时工具支持洋葱皮式的对照预览和范围循环播放，方便你在标区间时就发现循环不连贯的问题，而不是等导出后才发现。
  • 帧不允许重排、只能尾部追加+中间删除：每个视频源下的帧管理有这个限制，规划抽帧顺序时要提前想好，不要指望导入完还能任意调整帧的先后顺序。
  • 翻转不改原始帧：需要角色朝另一个方向时，可以对单帧或整个状态做水平翻转，工具只在预览/导出时生成临时翻转副本，不会破坏原始帧数据，比返工重新拍摄省事得多。

命名与登记的进阶讲究​

  • 状态命名要和场景字段严格一致：视频转图集里起的状态名（如 idle_dock、walk、bow），要跟场景面板里 NPC 的初始状态、巡逻状态字段填的字符串逐字符一致，大小写、下划线都要对上，否则游戏里会卡在第一帧或播放默认状态。
  • 锚点固定在脚底中心：所有导出的动画都以脚底中心为锚点，与游戏内精灵实体的对齐方式一致，这也是为什么角色贴地效果依赖这里的抽帧质量。
  • 动画浏览的图集字段是只读的：核对状态没问题后，要修改动画内容（哪怕只是删一帧）都得回视频转图集重新导出，不能在动画浏览面板里直接改。

导出与批量技巧​

  • 分别导出 vs 合并导出：每个动画可以单独导出成一张小图集，也可以把多个动画合并导出到一张大图集里（每个状态各自独立的描述信息）。合并导出还支持给不同动作clip设置不同的缩放比例，适合给「特写用的慢动作」单独提高精度。
  • 每行独立参数：导出工作台里，添加到导出列表的每一行（每个动画）都可以单独设置缩放、边距、世界尺寸，不是整批统一设置，方便给不同动作定制导出规格。
  • 删帧前的引用检查：工具会在你删除某一帧前，自动扫描所有动画序列里是否还有引用这一帧，并提示确认——这能避免你误删了一个正被某个动画序列使用的关键帧。

和其他工具/面板怎么配合​

  • 动画后处理：如果你已经有大批量、已经过稳定化处理的视频素材，全自动管线（动画后处理）可能比逐帧手动抽帧更省时间；本工具更适合精修关键角色或分支多的动作。
  • 动画浏览：核对状态列表的第一站，发现缺状态时要回视频转图集补一段区间，而不是在场景里硬填一个不存在的状态名。
  • 动画预览：比动画浏览的小窗更适合细验脚点贴地、循环衔接、滑步等问题，与游戏渲染规则完全一致，所见即所得。
  • 角色登记：动画包最终要挂到这里的角色身份上，场景里的 NPC 才能引用到对应动画。

危险区与边界​

  • 动画浏览面板里的图集字段是只读的——主编辑器无法直接修改动画内容，任何调整（哪怕只想删一帧、改一下帧率）都必须回视频转图集或动画后处理重新导出、重新绑定。
  • 覆盖旧图集有风险：场景里已经绑定的动画包，被新的导出结果覆盖后会直接生效——如果有别人正在改这个角色的场景内容，或者你还没确认新动画没问题，先备份旧图集再导出新的。
  • 只导出不绑角色 = 白做：动画浏览能看到新包，但场景里的 NPC 还是要在角色登记里手动指向新包，才会真正生效。
  • 更系统的哪里改了会丢，参考 危险区。
