导入一张素材

你手头有一张雾津用的图——关二狗的立绘、茶馆背景、音效文件——得先入库，游戏和编辑器才认。这一页走标准流程：分类导入 → 资源浏览器核对 → 主编辑器引用 → 运行预览确认。

这是什么（30 秒看懂）​

把这件事想成给城隍庙的档案库交新卷宗——你不能把画师给的图随手塞进任何抽屉，得先看它是「立绘」还是「背景」还是「音频」，归到对应的柜子，档案库才知道下次去哪个抽屉找它。分类导入就是这道「按柜归档」的工序：选文件、选类别、确认落位。归档之后，编辑器各面板挑图的下拉列表里才会出现它；如果绕过这道工序直接把文件丢进工程目录，编辑器根本不知道这张图存在。

这也是所有素材类教程的起点——不管你后面要把图挂到对话、场景，还是要把视频做成动画、给场景加深度，第一步永远是先让这张图或这段视频"入库"。

读完这页你能：

  • 用分类导入把外部图片放进游戏资源库。
  • 选对分类（立绘、背景、头像、音频等）。
  • 用资源浏览器核对文件落位是否正确。
  • 在主编辑器里把图挂到对应用途（对话、场景、档案等）。
  • 在游戏里看到图显示正常。

入门：手把手做第一次​

第 1 步：打开分类导入​

在游戏仓库根目录：

./dev.sh asset-ingest

也可以 ./dev.sh console 打开 Web 控制台，点「资源入库」按钮——效果一样。

窗口里会看到若干预设分类，每张图必须归到一类，系统才知道放进资源库的哪个位置。

分类典型用途
游戏 / 立绘NPC 对话大图、插图
游戏 / 背景场景背景图
游戏 / 角色头像对话框小头像
游戏 / 音频音效、语音片段
立绘可以分子文件夹

只有「立绘」分类允许按角色建子文件夹，方便归类。其它分类不要自建子目录，工具按约定结构去找文件，自建目录会让别处找不到。

第 2 步：拖入并导入​

  1. 把图片文件拖进导入窗口（或点浏览选取），可以一次选多个文件。
  2. 选中对应分类。
  3. 默认是复制进资源库；若勾选「移动」，原位置的文件会被移走——不确定就用复制，保留一份原始文件在别处备份。
  4. 点导入，等完成提示。

导入成功后，图已经在游戏运行时资源里，编辑器挑图时能从列表里看到。

操作示意​

第 3 步：用资源浏览器核对落位​

./dev.sh asset-browser

导入完不代表万事大吉，先开资源浏览器确认：

  1. 左侧选对应资源大类，展开子目录。
  2. 找到刚导入的文件，看预览区图是否清晰、方向是否正确。
  3. 确认文件名与预期一致——这一步能提前发现「导错分类」「文件损坏」等问题，比等到编辑器里报错「找不到资源」再回头查省事得多。

第 4 步：在主编辑器里引用​

./dev.sh editor

图进库以后，在哪引用取决于用途：

你想干什么去哪改
NPC 说话时旁边出立绘场景面板 → 选中 NPC → 立绘相关项
过场或对话里弹出一张图图对话或过场面板 → 动作里选「显示图片」
档案、见闻录里插图档案面板 → 富文本里插入图片（有专用按钮）

热区：场景里玩家能走近互动的区域；立绘多半挂在 NPC 或热区上，不是随便丢在场景顶层。

图片在资源库里有一个短名——编辑器列表里显示的路径名，运行时靠它找文件。选图时从下拉列表挑即可，不必手打路径。

第 5 步：运行预览​

F5 起运行预览，走到会显示这张图的位置：

  • 立绘：触发对应对话，看侧边图对不对、裁切是否正常。
  • 背景：走进场景，看比例与色调。
  • 档案：打开对应条目，看图是否在正文里。

图糊了或太大？可再用 ./dev.sh image-resizer 做等比缩放或镜像，或重新导一张合适尺寸的。详见 图片缩放工具。

流程示意​

雾津完整实例：李天狗新表情立绘​

美术给你一张李天狗新表情立绘，要在他和关二狗斗嘴时出现：

  1. ./dev.sh asset-ingest，分类选「游戏 / 立绘」，可放进立绘子文件夹里标清角色名，比如 li_tiangou/smirk.png。
  2. 导入完开 ./dev.sh asset-browser，找到这张图，确认预览清晰、没有裁切错位。
  3. 主编辑器 → 场景或角色登记，把李天狗的立绘指到新图。
  4. 图对话里若有专用立绘切换动作，一并改掉，让斗嘴那几句台词切到新表情。
  5. F5 进游戏，触发那段斗嘴，确认新表情亮出来、方向和裁切都对。

图进了库、挂对了人，戏才接得上。

进阶：每一项都讲透​

分类怎么选、选错会怎样​

  • 分类不是随便贴的标签，它决定文件落到工程的哪个目录，进而决定哪些面板的下拉列表里能找到它。类别选错最直接的后果是：编辑器对应面板的下拉里根本看不到这张图，好像「导入了但消失了」。
  • 只有立绘分类允许建子文件夹（按角色分类整理），其它分类（背景、头像、音频）不要自建子目录，工具按约定路径查找，自建目录容易导致资源浏览器和面板都找不到。
  • 拿不准该选哪个分类时，先想清楚这张图的最终用途——是要挂在 NPC 身上说话时弹出来（立绘），还是整张背景（背景），还是对话框头像小图（头像）。

复制 vs 移动​

  • 默认是复制：外部原文件留在原处，工程里多一份拷贝。这是更安全的默认选项，尤其是原文件还在其它协作流程里被引用时。
  • 勾选移动后，原始位置的文件会被移走——只有在你确定这份文件以后只归工程管、不会被别处引用时才用，否则容易破坏画师/音效师本地的工作目录结构。

覆盖策略​

  • 导入前工具会预览目标路径，同名覆盖要格外小心：覆盖前确认没有场景、角色仍绑着旧文件——覆盖后旧引用会直接变成新图，如果新旧图尺寸、构图差异大，可能造成场景走位或立绘裁切错乱。
  • 想保留旧版本做对比时，导入成新文件名（加版本后缀），而不是直接覆盖，等确认新图没问题再清理旧文件。

资源浏览器的进阶用法​

  • 资源浏览器不只是「看一眼」，它还承担着排障入口的角色：主编辑器报错「找不到资源」时，第一反应应该是回资源浏览器查路径和文件名是否一致，而不是直接怀疑编辑器坏了。
  • 两套立绘候选可以在资源浏览器里并排预览对比，选定更贴合雾津气质的那套再去角色登记里绑定，比先绑好再切换更省事。
  • 资源浏览器只读、只看，不改游戏数据表——它不会帮你完成「登记引用」这一步，浏览完了该去主编辑器该去的面板，还是要手动挂上。

尺寸不合适时的处理​

  • 背景图分辨率远超游戏实际需要时，先用 图片缩放工具 等比缩到目标宽度再入库，能减小最终包体。
  • 同一张立绘只有朝右版本，但某个场景需要角色朝左站位时，可以用图片缩放工具的水平镜像功能直接出一份朝左的副本，不需要让画师重画一次；镜像后要注意游戏内角色锚点通常在脚底，镜像后走位效果要在预览里重新验证一遍。
  • 图片缩放工具做的都是非破坏性处理，一律导出新副本，不会覆盖你手里的原图，可以放心反复尝试不同尺寸。

和其他工具/面板怎么配合​

  • 视频转图集：如果拿到的是一段动作视频而不是静态图，第一步同样是先分类导入原始视频素材，再开视频转图集处理，见 把视频做成角色动画。
  • 场景深度工具：背景图入库、场景绑定后，才轮到场景深度工具去生成深度和碰撞，见 给场景加遮挡/深度。
  • 富文本：档案、见闻录等富文本框里插入的图片标签，需要图片已经入库并有短名才能正常显示，具体标签写法见 怎么写带引用的文本。

批量导入的效率窍门​

  • 协作同事发来一个 zip 包时，先解压再分批导入，每批固定一个类别，比混着选来选去更不容易选错分类。
  • 一次收到一批新美术/音频素材时，第一时间分类导入，别让文件散落在下载夹里——积压越多，后面回忆「这张图该归哪类」的成本越高。
  • 替换旧版立绘时，先导入新文件（不覆盖），在主编辑器里逐个确认引用改对以后，再回头清理旧文件，比直接覆盖更安全。

危险区与边界​

  • 分类导入、资源浏览器本身不涉及游戏数据表的重建区风险，但同名覆盖是不可逆操作：覆盖前没做好确认，旧文件就真的没了，除非你在别处（DVC、备份）另存了一份。
  • 图片文件本身不属于编辑器的「重建区」，但要留意改磁盘上的文件名这件事——已经被面板引用的文件改名或换路径，会导致引用断链，改名前先在资源浏览器或全局搜索确认没有别处还在用旧文件名。
  • 更系统的「哪里改了会丢」说明见 危险区。

常见问题​

现象原因怎么办
导入完编辑器下拉里看不到分类选错，文件落到了别的目录回分类导入重新选对分类导一次
立绘方向反了只有一个朝向的原图，场景需要另一朝向用图片缩放工具做水平镜像出副本
编辑器报错「找不到资源」文件名或路径和引用不一致开资源浏览器核对实际文件名与路径
覆盖后场景走位或裁切乱了同名覆盖，新旧图尺寸/构图差异大换新文件名重新导入，逐个核对引用
音效导入了但游戏里没声音只入库未在音频面板或动作里登记引用回音频面板或对应动作补上引用
背景图太大、加载卡顿原图分辨率远超游戏实际需求用图片缩放工具等比缩小后重新导入
