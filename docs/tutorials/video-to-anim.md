# 把视频做成角色动画

雾津街上的人物不能只会站桩——走路、挥手、蹲点查案，都得有**动画**。若你手里是一段拍好的或生成的角色视频，这一页教你变成游戏里能播的精灵动画：抽帧 → 拼图集 → 登记状态 → 预览 → 进场景验证。

---

## 读完你能做到什么

- 用**视频转图集**从视频里抽出动作帧
- 导出图集与动画描述，供游戏加载
- 在主编辑器**动画浏览**里核对各状态（站立、行走等）
- 用**动画预览**和运行预览确认角色动起来

---

## 整条管线长什么样

```mermaid
flowchart LR
  A[角色动作视频] --> B[视频转图集 · 抽帧拼图]
  B --> C[动画浏览 · 核对状态]
  C --> D[角色登记 · 挂动画包]
  D --> E[场景里看到角色动]
  E --> F[动画预览 / F5 细验]
```

> **图集**：把很多小帧拼成一张大图，游戏按格切出来播——省资源、加载快。

---

## 第 1 步：打开视频转图集

这个工具从主编辑器菜单进（没有单独的 `./dev.sh` 命令）：

```bash
./dev.sh editor
```

菜单栏 → **工具 → 外部工具（新进程）** → 选 **视频转图集**。

也可以 `./dev.sh console`，在工具列表里找同名入口。

---

## 第 2 步：建工程、抽帧

1. 新建或打开一个**工作区**（工具会记住上次进度）
2. 导入视频——例如关二狗在码头蹲点张望的片段
3. 在时间轴上标出要用的**区间**（哪几秒算「走路」，哪几秒算「站立」）
4. 预览抽出来的帧，删掉糊的、重复的
5. 为每个动作起名：常见有 idle（站）、walk（走）、run（跑）

多段视频可以并进同一个角色工程，各管各的状态。

## 第 3 步：导出

点**导出**，工具会生成：

- 一张**图集大图**（所有帧格排在一起）
- 一份**动画描述**（哪几格、多快、循不循环）

导出位置按角色/动画包归好类，方便后面登记。主编辑器里**动画浏览**面板的图集字段是只读的——要换图集，回这个工具重新导。

---

## 第 4 步：动画浏览里核对

```bash
./dev.sh editor
```

左侧 → **资源与本地化 → 动画浏览**：

1. 找到刚导出的动画包
2. 看各**状态**的帧序、帧率、是否循环
3. 世界尺寸不对可在面板里微调（图集本身仍须回视频转图集改）

想单独、快速地看某个角色某状态：

```bash
./dev.sh anim-preview
```

浏览器里选角色和状态，所见与游戏里一致。

---

## 第 5 步：挂到角色、进场景

1. **角色登记**面板 → 选中角色（如关二狗）
2. 把**动画包**指到刚导出的那份
3. **场景**面板 → 场景里的 NPC 选同一角色身份
4. **F5** 运行预览，看走路、站立是否顺滑、脚底是否贴地

脚底飘、比例怪？多半是抽帧区间或世界尺寸要再调一轮。

---

## 雾津小例子

要给**城隍庙**前卖糖画的师傅补一套「招手揽客」动画：

1. 外部工具打开**视频转图集**，导入招手视频，标区间，状态名 `wave`
2. 导出后进**动画浏览**，确认 `wave` 循环自然
3. **角色登记**里师傅的动画包加上 `wave`
4. 场景里师傅 NPC 走近能对话；若还要自动招手，在区域或 NPC 行为里编排播放该状态
5. `./dev.sh anim-preview` 先看，再 **F5** 进城隍庙验

人动起来，街才活。
