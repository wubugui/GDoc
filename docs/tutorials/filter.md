# 调一个画面滤镜

同一座码头，日头下和雨夜里该是两种气色。**滤镜**就是盖在画面上的一层色调——不改变场景里摆了啥，只改玩家眼睛里的氛围。这一页：调出你要的色 → 存成预制 → 挂到场景 → 预览。

---

## 这是什么（30 秒看懂）

把滤镜想成戏台上罩在灯泡外面的**一层彩色玻璃纸**——舞台上的桌椅道具（场景里摆的东西）一件没动，但换一张玻璃纸，观众看到的整个氛围就变了：暖黄的纸让人觉得是黄昏庙会，青灰的纸让人觉得是雨夜孤灯。滤镜工具就是帮你调这张"玻璃纸"颜色的地方——调色相、饱和度、亮度、对比，存成一个预制，挂到某张场景上，玩家一进场景就自动罩上这层色调。

滤镜和"场景里放了什么"是两条完全独立的线：换滤镜不会移动任何一个热区、NPC 或背景图的位置，只改变整体画面的色彩观感。

读完这页你能：

- 用**滤镜工具**可视化调色、对比前后效果。
- 导出与游戏一致的滤镜预制。
- 在主编辑器**滤镜**面板或**场景**里挂上滤镜。
- 运行预览里确认氛围对味，且不会把画面调得看不清人脸。

---

## 入门：手把手做第一次

### 滤镜是什么

> **[滤镜](../reference/glossary)**：一组色彩变换参数，让画面偏暖、偏冷、压暗、提对比等。场景可以指定「进这个场景时用哪个滤镜」。

滤镜工具和主编辑器的**滤镜**面板看的是**同一批**预制——两边改一处，另一边刷新就能看见，不存在"两份数据要手动同步"的问题。

### 第 1 步：打开滤镜工具

```bash
./dev.sh filter-tool
```

也可以从 `./dev.sh console` 的 Web 控制台点「滤镜工具」，或在主编辑器菜单 **工具 → 外部工具** 里找到「滤镜工具」入口。

窗口里通常是：左边调参滑条与预制列表，中间实时预览，右边可对比前后效果。

### 第 2 步：调参、存预制

1. 从空白或相近预制开始——雨夜可先选偏冷的底子；工具还内置了几套现成预制（还原、夜晚、黄昏、阴天、复古），可以直接从这些起步微调，不必每次从零开始。
2. 拖滑条调**色相、饱和度、亮度、对比**等，边拖边看预览。
3. 满意后**保存**为新预制，起个认得出的名字，如 `wujin_rain_night`——这个名字将来就是滤镜的 **id**，保存之后**不能改名**（想改名只能新建一份、场景那边重新绑定）。
4. 预制会自动进游戏数据目录，主编辑器能读到。

:::tip[先想场景再调参]
心里先钉住「这是哪一场戏」——城隍庙夜祭偏暖黄，义庄阴风偏青灰——再动手，少返工。
:::

### 第 3 步：挂到场景

```bash
./dev.sh editor
```

两种方式：

| 方式 | 做法 |
|---|---|
| 场景级 | **场景**面板 → 选中场景 → 找「滤镜」项 → 下拉选刚存的预制 |
| 仅查阅 | **规则与经济 → 滤镜**面板浏览、改名备注（编号通常跟文件名走，别乱改） |

场景上的滤镜 = 玩家**一进该场景**就套上的色调。过场里若还要临时变色，在过场面板用专门步骤（和场景滤镜是两条线）。

### 第 4 步：运行预览

**F5** 走进挂了滤镜的场景：

- 整体色调是否符合这场戏。
- 和相邻场景切换时会不会太跳。
- UI、立绘是否还能看清（别调太狠）。

## 雾津完整实例：义庄夜访

**义庄**夜里要有「风冷骨寒」感，但不要把对话立绘压成一片黑：

1. `./dev.sh filter-tool` 新建 `yizhuang_night`，略降亮度、偏青、对比稍提。
2. 满意后保存为预制，`yizhuang_night` 这个名字之后不能改。
3. 主编辑器 **场景** → 义庄 → 滤镜选 `yizhuang_night`。
4. **F5** 夜探义庄，站定看背景；再触发一段对白，确认立绘仍清晰、不至于糊成一团黑影。
5. 若过场有闪回，可在过场步骤里另加短时滤镜变化，与场景默认滤镜配合，做出「回忆瞬间褪色」的效果。

灯色对了，玩家才觉得冷的是义庄，不是屏幕。

## 进阶：每一项都讲透

### 参数逐条讲透

- **色相**：整体色彩偏移方向，偏暖（黄橙）或偏冷（青蓝），是营造场景基调最直接的一个参数。
- **饱和度**：颜色的鲜艳程度，回忆、闪回类场景常常调低饱和度做「褪色」效果。
- **亮度**：整体明暗，夜景、阴森场景常压暗，但压得太狠会让立绘细节看不清。
- **对比度**：明暗反差，适度提高对比能让阴影更有层次感，调过头则容易死黑死白。
- 这几个参数背后对应的是一组完整的色彩矩阵（20 个矩阵元）加一个透明度值，滤镜工具的滑条只是把这套矩阵变得可视化好调，你不需要，也不应该去手改数值本身。

### id 与预制管理

- 滤镜保存后 **id 与文件名绑定，只读不可改名**——这是本页最重要的边界提醒。想换个名字，唯一办法是新建一份新滤镜，再回所有绑定过旧滤镜的场景/过场，批量换绑到新 id，然后再删掉旧的。
- 自定义预制会存一份在工具本地的列表里，方便下次继续调；但**真正生效的是保存进工程滤镜目录的那份**，换机器或换协作者之前，确认你要用的预制已经存进工程，而不是只留在本地列表里。
- 内置的几套预制（还原、夜晚、黄昏、阴天、复古）可以当作起点直接复制修改，不必每次从中性开始摸索参数方向。

### 场景级 vs 过场级滤镜

- **场景级滤镜**：玩家一进入这张场景地图，就自动套上这个色调，直到离开场景或场景本身换绑其它滤镜。这是最常用、最持久的挂载方式。
- **过场级滤镜变化**：过场演出里可以临时切换滤镜做瞬时效果（比如打雷瞬间、闪回进入），这是和场景默认滤镜**平行的另一条线**，两者互不覆盖也互不依赖——过场结束后画面通常会回到场景自身绑定的滤镜。
- 设计闪回、回忆类内容时，常见做法是给回忆专门做一条**独立的褪色滤镜**，不要和日常黄昏共用一条，否则玩家分不清「现在是黄昏」还是「这是一段回忆」。

### 和光照、场景深度的配合

- 滤镜管的是**整屏色调**，[光照体积烘焙](../editors/render-domain/lightvolume-lab) 管的是**局部光影**（比如某盏灯周围的光晕），两者叠加使用时要小心过曝或过暗——义庄这类阴森场景常见的坑是滤镜已经压得很暗，再叠一层暗色光照体积，结果玩家什么都看不清。
- 滤镜完全不涉及**遮挡与碰撞**——角色走位穿模的问题不归滤镜管，那是 [场景深度工具](../editors/render-domain/scene-depth-editor) 的职责，两者是完全独立的两条工序，别混淆排查方向。

### 批量与效率窍门

- **一个预制多场景复用**：如果整个游戏想统一一种"晨雾"基调，做一个预制、多张场景共用即可，不需要给每张场景单独做一份几乎一样的滤镜。
- **从相近预制起步**：内置预制或已有的自定义预制都可以当底子复制修改，比每次从中性矩阵开始调色更快找到方向。
- **对比预览别偷懒**：调完参数后，别只看滤镜工具自己的预览小窗就下定论，一定要回主编辑器挂到真实场景、用运行预览走一遍——尤其要顺带触发一段对话，检查立绘和 UI 在这个滤镜下是否还清晰可读。

## 危险区与边界

- **滤镜 id 保存后不可改名**：这是唯一需要特别注意的边界，改名需求只能走"新建 + 批量换绑 + 删旧"的流程，不存在原地重命名的操作。
- **删除滤镜前**要确认没有场景还挂着这个 id——删除后，场景那边的滤镜下拉会变成空引用，容易被误认为是 bug。
- 滤镜数据结构本身简单，不属于主编辑器的高危重建区，日常调参数、改数值都可以放心操作。
- 更系统的说明见 [危险区](../editors/concepts/danger-zone)。

## 常见问题

| 现象 | 原因 | 怎么办 |
|---|---|---|
| 场景色调毫无变化 | 场景没选滤镜，或选错了预制 | 回场景面板核对滤镜绑定字段并重新保存 |
| 角色脸糊成一片、看不清表情 | 矩阵调得过激，或亮度压得太低 | 降低强度，往中性方向回调，适度即可 |
| 想改滤镜名字改不了 | id 与文件名绑定，只读 | 新建一份滤镜，场景批量换绑后再删旧的 |
| 义庄这类夜景和鬼打墙叠加太脏 | 滤镜与光照体积双重偏色叠加 | 分工明确：光照管局部氛围，滤镜管整体色调，各退一步 |
| 删除滤镜后场景异常 | 仍有场景绑着已删除的滤镜 id | 删除前先解绑或换绑所有引用场景 |
| 回忆闪回和日常黄昏看着一样 | 两者共用了同一条滤镜 | 给回忆单独做一条独立的褪色滤镜 |
