# 出问题怎么办

油灯还亮着，书案却乱了——游戏起不来、改完没反应、做好的东西一时没找到、担心手改的内容一保存就没了。这一页按**用户嘴里常说的四类事**列自查步骤；仍解决不了，再带着现象找懂工程的人。

---

## 读完你能做到什么

- **起不来**：环境、命令、端口怎么排
- **改了没生效**：保存、预览、触发条件怎么查
- **东西不见了**：引用、条件、位面、登记漏了啥
- **担心保存丢内容**：哪几个字段编辑器改不到、哪几个操作点了会有后果

---

## 总流程：先别慌，按顺序查

```mermaid
flowchart TD
  Q[出问题了] --> A{游戏能起吗?}
  A -->|不能| B[起不来 · 第一节]
  A -->|能| C{改完看得到吗?}
  C -->|不能| D[改了没生效 · 第二节]
  C -->|能但缺东西| E[东西不见了 · 第三节]
  D --> F{保存后内容没了?}
  E --> F
  F -->|是| G[保存丢内容 · 第四节]
  F -->|否| H[./dev.sh validate-data]
```

## 一、起不来

### 现象

- 敲 `./dev.sh` 报错
- 编辑器窗口闪退或打不开
- 游戏浏览器一片白、连不上

### 自查清单

| 步骤 | 做什么 |
|---|---|
| 1 | 命令是否在**游戏仓库根目录**跑的？文档站目录里跑一律不对 |
| 2 | 是否跑过 `./bootstrap.sh`？报「运行环境缺失」就先 bootstrap |
| 3 | 资源是否拉过？`./dev.sh pull` |
| 4 | 起游戏用 `./dev.sh game start`，不是别的拼写 |
| 5 | 浏览器地址跟终端提示的端口一致吗？5173 被占时会换端口 |
| 6 | 编辑器打不开：关掉旧窗口再 `./dev.sh editor` 重开 |
| 7 | 专项工具（视差编辑器、场景深度等）打不开：多半是**端口被占用、复用了旧实例**——看终端提示的实际端口，关掉旧窗口重开 |
| 8 | 不确定该开哪个工具：`./dev.sh console` 打开 Web 控制台，按钮里逐个找 |

### 雾津例子

新机器第一次编雾津，直接 `./dev.sh editor` 报错——先 `cd` 到游戏仓库，`./bootstrap.sh`，再 `./dev.sh pull`，然后一切就顺了。

## 二、改了没生效

### 现象

台词改了、图换了、物品加了，进游戏还是旧的。

### 自查清单

| 步骤 | 做什么 |
|---|---|
| 1 | **保存了吗？** Ctrl+S / Ctrl+Shift+S，标题栏无未保存标记 |
| 2 | **预览对了吗？** 内嵌预览用 **F5**；只开浏览器时确认已刷新，或 Shift+F5 再 F5 |
| 3 | **改的是玩家会走到的那条吗？** 对白改错图、任务指错 id，都会像「没改」 |
| 4 | **触发条件满足吗？** [旗标](../reference/glossary)、任务进度、[位面](../reference/glossary)不对，内容不会出现 |
| 5 | **跑校验** `./dev.sh validate-data`，有错误先修 |
| 6 | **缓存错觉** 独立窗口预览关干净再开 |
| 7 | **结构性改动没跟上** 新场景、新任务链、新建小游戏关卡，热重载有时跟不上——`Shift+F5` 完整停掉再 `F5` 重开 |
| 8 | **面板没单独保存** 有的内容分散在多个面板（比如任务和物品），只存了一个，另一个还是旧的——习惯性 `Ctrl+Shift+S` 保存全部 |

详见 [用运行预览验证改动](./preview-verify)。

### 雾津例子

城隍庙对白改了但游戏里还是旧词——发现改的是备用分支图，玩家走的是主线节点；换对节点保存 F5 就好。

## 三、东西不见了

### 现象

NPC 不露头、热区点不到、物品买不到、小游戏进不去、档案不解锁、声音没响。

### 常见原因

| 原因 | 你怎么查 |
|---|---|
| **条件未满足** | 图对话、热区、NPC 上的条件——任务未到、旗标未立 |
| **位面不对** | 东西挂在非常规位面，玩家还在默认现实世界 |
| **没登记全** | 物品没进商店价目、小游戏没建实例、音频没挂场景 |
| **引用断了** | 动作里指的任务、关卡、物品编号和面板里不一致 |
| **场景入口没了** | 转换热区删了、坐标偏到墙外 |
| **资源没入库** | 图、音频没走分类导入，列表里选不到 |
| **小游戏 id 不一致** | 索引里的 id 和实例内部的 id 必须完全一样，不一致就打不开或开错关 |
| **小游戏 spot 对不上** | 水域等挂在场景交互点上的小游戏，交互点 id 和场景里配的位置不一致，点了没反应 |
| **档案条目一直灰着** | 解锁条件没满足，或条件本身填错 |
| **档案读过没留痕** | 首次阅读动作没配，游戏侧条件判断仍当没读过 |
| **音效登记了却听不到** | 场景/过场/信号 Cue 里引用的 id 和音频面板登记的 id 对不上 |

不是「丢了」，多半是**链断了一环**——从玩家进场景到触发的整条链，用预览走一遍，缺哪步补哪步。

### 雾津例子

香烛铺买不到新香——价目表有货，但场景里进铺热区被误删；回场景面板补热区，F5 再买就有。

## 四、担心保存丢内容（危险区）

### 现象

手改过的字、塞进去的额外说明，开面板再保存怀疑是不是没了；或者担心音频音量写了会白写。

保存很稳：你填的会存住，你没碰的字段也不会被清掉。真正要留意的，只有下面这**两类**。

### 一、真正的盲区（编辑不到，但不会丢）

这些字段游戏运行时真的会用，但主编辑器对应面板没有入口——**不会被丢**，只是编辑不了，得换地方：

| 够不着的东西 | 去哪改 |
|---|---|
| 场景多层背景、深度/遮挡碰撞主体 | [场景深度工具](../editors/render-domain/scene-depth-editor) |
| 玩家化身外观细节 | [玩家化身面板](../editors/panels/avatar) |
| 旗标的「迁移」「运行时」块、实体像素密度匹配 | 暂无界面，手改或找维护者 |

### 二、明确点了才会触发的操作

这些不是悄悄丢，是你主动做了某个动作才有的后果：

| 操作 | 后果 |
|---|---|
| 区域从「标准」改成「深度地板」 | 会清掉它的进入/停留/离开动作和气味，切回来也找不回 |
| 场景面板「重新导入背景」 | 多层背景会被压成单层 |
| 删小游戏实例 | 登记表里没了，但磁盘上的旧实例文件不会自动删，需手动清 |
| 转盘/扎纸的专家 JSON、裸输入字段格式不合法 | 会导致这一次保存**明确失败**（不是静默丢数据），动手前留意格式 |

### 怎么用这页判断

1. 你担心的那个东西，面板里有对应的框吗？有就放心——填的会在，没碰的也会保留。
2. 编辑不到的字段，对照上面第一类找到该去的专项工具。
3. 踩到第二类里明确的破坏性操作，那是设计使然，不是 bug。
4. 拿不准就去 [危险区速查](../editors/concepts/danger-zone) 核对逐面板清单。
5. 重要文案还是建议用版本管理，大改前先备份或提交一版，图个安心。

### 雾津例子

给调查热区手写了一大段说明，回来一看还在——面板里填的东西都会保留；真正要小心的是把区域改成深度地板，或者用场景面板的「重新导入背景」换了一张多层背景图。

## 快速对照表

| 用户说法 | 先看 |
|---|---|
| 起不来 | bootstrap → pull → 正确目录与命令 |
| 改了没生效 | 保存 → F5 → 对错节点 → 条件 |
| 东西不见了 | 条件 · 位面 · 登记 · 引用链 |
| 担心保存丢内容 | 危险区 · 多数其实不会丢，先查真正的盲区和明确操作 |
| 小游戏开不了/开错 | 索引 id 与实例 id 是否一致 |
| 档案不解锁/读了没留痕 | 解锁条件 · 首次阅读动作 |
| 音频没声音/听不到 | 资源是否入库 · id 是否对得上 |

---

## 仍解决不了时

带上这些信息找人效率最高：

1. 你做了什么（改哪个面板、什么内容）
2. 期望什么、实际什么
3. 终端报错原文（若有）
4. `./dev.sh validate-data` 是否报错
5. 是否动过编辑器面板外的手写数据
