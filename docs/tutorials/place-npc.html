放一个会说话的 NPC

雾津的码头、茶馆、义庄——少了会搭话的 NPC，就像少了人声。这一页教你在场景里放一个 NPC，绑上一张对白图，让玩家走近按互动键就能开口。

这是什么（30 秒看懂）​

NPC 是场景里一个有血有肉站着的角色——他有自己的位置、朝向、动作，还能巡逻。和「热区」不一样：热区是一块能点的地方，NPC 是一个「人」。你在场景画布上新增的这个 NPC，其实是把角色登记里那份「关二狗长什么样、有哪些动画」的档案，实例化成雾津街头这张地图上具体站着的一个人——就像折子戏里，演员表上有关二狗这个角色，但每一幕布景里，他具体站在台上哪个位置、朝着哪边，是这一页管的事。

放好 NPC 之后，光站着不够——要让他能「开口」，还得把他和一张 对白图 绑在一起，指定「玩家走近按互动键时，从哪句台词开始念」。

读完你能做到什么​

  • 在场景画布上新增 NPC 并拖到合适位置
  • 给 NPC 起名、绑对白图与入口节点
  • 设互动距离与面向
  • 运行预览里走过去跟 NPC 说话

怎么开工具​

主编辑器 → 物理世界 → 场景（NPC 是场景的一部分）

对白内容在 叙事编排 → 图对话 里编。两扇面板常来回切。

./dev.sh editor

手把手逐步操作​

第 1 步：准备对白（若还没有）​

  1. 打开 图对话
  2. 选或新建一张对白图（见 改你的第一句对白）
  3. 至少放：台词节点 → 结束
  4. 记住图的名字，以及第一个台词节点的入口名（检查器里「入口」或列表第一项）

第 2 步：打开目标场景​

  1. 场景面板 → 场景列表选「雾津街头」等
  2. 画布显示当前场景

第 3 步：新增 NPC​

  1. 列表面板或工具栏点 新增 NPC
  2. 画布上出现 NPC 标记，拖到要站的位置——比如关二狗蹲在茶馆后门
  3. 右侧检查器填写：
项说明
标识系统用，英文或拼音，别跟别的 NPC 撞名
显示名玩家看到的名字，如「关二狗」
对白图下拉选刚准备好的那张图
对白入口从这张图的哪个节点开始聊
互动距离玩家多近能按 E 说话
初始面向左/右，决定立绘或精灵朝向
镜头缩放对话时镜头推近程度（可选）

第 4 步：动画与碰撞（可选）​

  • 动画文件 / 初始动作：让 NPC 站着别发呆（动画在资源管线里备好）
  • 碰撞多边形：在画布上拖点，框住可走/不可走的边界

若 NPC 只在特定任务阶段出现，在检查器里加条件；条件用法见 怎么设条件。

第 5 步：保存并验证​

  1. Ctrl+S 保存场景
  2. F5 运行预览
  3. 控制关二狗（或玩家）走到 NPC 旁，按 E 或 空格
  4. 对白框应弹出，台词来自你绑的对白图

流程示意​

另一种做法：用热区挂 NPC​

场景里还有一种 NPC 型热区——不占独立 NPC 槽，适合「调查点顺便聊两句」。做法：

  1. 场景里 新增热区，类型选 NPC 对话
  2. 指定引用哪个已登记的 NPC 身份
  3. 配动作或对白图（视检查器提供的模式）

独立 NPC 实体适合长期站在地图上的角色；热区型适合一次性或跟物件绑定的对话。详见 场景面板。

雾津完整实例​

任务：在雾津街头放李天狗，玩家一靠近就能问寻狗的事；李天狗只在玩家已经进过城隍庙之后才出现在这个位置，之前这里空无一人。

  1. 图对话里备好「李天狗_问路」对白图，入口是第一句「道士穷，狗也穷？」
  2. 场景选雾津街头，新增 NPC，标识 li_tiangou_street，显示名「李天狗」
  3. 绑对白图「李天狗_问路」，入口选默认第一句
  4. 拖到土地庙台阶旁，互动距离略大一点（玩家不用贴脸）
  5. 检查器里加条件：旗标「已进城隍庙」为真
  6. F5：还没进庙时，这个位置站的应该是空的；进庙一趟回来后，走过去搭话——雾里多一个能回嘴的道士

常见卡点​

NPC 明明摆在画布上，游戏里却看不见？ 先看检查器里有没有挂了条件并且勾了「条件不满足时隐藏」——不满足就会完全不显示。再看有没有设位面归属：如果只勾了某几个位面，玩家当前所在的位面不在其中，NPC 也不会出现。

走到跟前按互动键没反应？ 多半是互动距离设得太小，或者对白图 / 入口没选对（选了图但没选入口，等于没指定从哪开始）。也可能是碰撞多边形挡住了玩家靠近的路径，走不到能触发互动的范围内。

NPC 说的是别的台词，不是我绑的那张图？ 检查这个 NPC 到底绑的是哪张对白图——同名 NPC 在不同场景里可能各自绑了不同的图，改错了地方很常见。

巡逻路线保存后变乱了？ NPC 的巡逻是「重建区」：保存时只保留路线、速度、移动动画这几项，如果你曾经期待巡逻里还带着别的自定义行为（比如站定说话），那部分不会被保留，需要另外用条件或热区实现。

碰撞多边形拖了但 NPC 还是能被穿过或者卡住？ 碰撞多边形是手动在画布上拖点围出来的，形状和实际立绘对不上是最常见的原因——回到画布放大看看多边形是否贴合站立的脚下范围，而不是整个立绘的外框。

进阶变体​

  • 位面归属限定出场：NPC 详情面板里的「位面归属」默认是「所有位面都在」，勾选具体几个位面后，NPC 只在这些位面存在——常用来做「同一张地图，不同位面看到不同的人」，比如阴阳两界各自的场景住客。
  • 条件 + 隐藏两件事分开配：条件本身只决定「能不能互动」，要不要在不满足时干脆连人影都不显示，是另一个开关（条件不满足时隐藏实体）。想做「远远能看到人影，走近才发现打不开话匣子」，就只设条件、不勾隐藏。
  • 对话镜头缩放做特写：重要 NPC 第一次登场，给「镜头缩放」设一个比平时更近的值，对话开始时镜头会推近，比一路平视镜头更有戏剧感——这个字段是可选的，留空就用场景默认镜头。
  • 巡逻折线做日常感：给 NPC 拉一条巡逻折线，让他不是死站着，而是在码头几个点之间来回走——适合背景气氛角色；真正重要的对话 NPC 一般还是定点，方便玩家找到。
  • NPC 型热区 vs 独立实体的取舍：一次性路人（问完路就走、剧情推进后消失）用热区挂 NPC更轻量，不用管巡逻、碰撞这些额外字段；长期活在地图上、值得让玩家反复回来聊的角色，用独立 NPC 实体更方便管理位置和状态。
  • 多个入口对应不同关系阶段：同一张对白图可以有不止一个可用入口，NPC 检查器里的「对白入口」决定这次绑的是哪一个——常见做法是同一张图里放「初次见面」和「已经熟识」两套开场，靠 NPC 侧或条件切换指向哪个入口，而不是为每个阶段各画一张新图。
