条件类型速查

条件决定「能不能 / 要不要」——任务能否接取、选项是否亮起、地图边是否可走。条件独立出现在各面板的条件槽里，不写进动作参数。

入门见 怎么设条件。

结构一览​

  • 叶子：查一项游戏状态。
  • 组合：把多条判断套在一起；可嵌套，但不宜过深（有深度上限，极端复杂请拆旗标或任务）。

七种叶子​

叶子判断什么怎么填什么时候用
旗标某个旗标当前值选或填旗标名；选比较符（等于、不等于、大于、小于等）；填比较值。值里可插富文本引用。通用进度开关：「已交香火钱」「好感≥3」
任务某条任务进度选任务；选状态：未接 / 进行中 / 已完成任务门控：「寻狗记进行中才能进侧门」
剧本某剧本线整体选剧本；选阶段名、阶段状态；可选结局多线叙事：「纸人线已到第二章」
剧本行剧本里某一行的节拍选剧本行；选：未触发 / 已激活 / 已完成细粒度节拍：「评书段已听完」
叙事状态叙事状态机是否到达某状态选叙事图；选状态；选「已到达」位面、章节大开关：「夜探状态已激活」
活计计数某条可重复活计累计完成次数选活计图 / 结算出口；选比较符；填次数「背尸送义庄做过 ≥ 2 单才开新闲话」
激活位面当前激活位面是否等于某 id选位面 id；「不是现世」用取反包一层「激活位面 = 现世」「鬼打墙位面激活时才显示某热区」

拼写必须与游戏里登记的名称一致。编辑器不一定当场报错——错了会静默不成立，改完务必预览两种状态各走一遍。

三种组合​

组合意思例子
全部成立每一条子条件都得满足任务进行中 且 旗标为真
任一成立至少一条满足即可持有道具甲 或 持有道具乙
取反子条件不成立时才算成立未完成某任务

分支节点的特殊口径​

图对话里「分支」节点有两种写法：

  • 结构化条件树：与上表七种叶子相同，功能最全。
  • 内联与写法（仅部分分支）：只支持旗标、任务、剧本三类简化判断。

新内容优先用结构化树；内联适合简单二分。

需求用条件用动作

危险

动作编辑器没有通用「如果…则…」框。条件只在外层条件槽或分支节点上设。

常见问题​

为什么我设了条件，选项还是一直亮着/一直灰着？ 最常见的原因是名字没对上——旗标、任务、剧本、位面的名字必须和登记的一模一样。逐字核对一遍，再预览试试。

「活计计数」和「任务状态」有什么区别？ 任务状态看的是这条任务当前是未接 / 进行中 / 已完成；活计计数看的是可重复活计历史上完成过几单，适合做「干过两趟背尸才开新台词」这类门槛。

「激活位面」怎么表达「不在现世」？ 用「取反」包一层「激活位面 = 现世」。

「全部成立」和「任一成立」能不能混着套？ 能。超过三四层建议改用一个新旗标把中间结果记下来。

同一个条件要在好几个地方判断，要不要抄好几遍？ 条件树本身不能跨面板共享，但可以让多处都判断同一个旗标或同一个任务状态。

验证清单​

  • 旗标/任务/剧本/位面名与登记一致
  • 用了「全部」还是「任一」符合策划意图
  • 保存后预览：满足与不满足各走一遍
  • 活计计数用的是累计次数，不是「当前是否激活」
