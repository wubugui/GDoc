# 富文本标签速查

玩家看到的对白、描述、档案正文里，用 **`[tag:类型:标识]`** 插入动态引用——运行时自动换成当前名字、数值或链接，改登记名全文跟着变。

入门见 **[怎么写带引用的文本](../editors/concepts/rich-text)**。

:::danger[写法必须带 tag 前缀]
正确：`[tag:item:狗哨]`、`[tag:npc:关二狗]`、`[tag:player]`  
错误：`[item:狗哨]`、`[npc:关二狗]`、`[player:name]`——少了 `tag:` 前缀，运行时不会按引用解析。
:::

---

## 八种引用标签

| 标签写法 | 引用什么 | 典型句子 |
|---|---|---|
| `[tag:string:分类:键]` | 文本库里的固定条目 | 系统统一称呼、格式化用语 |
| `[tag:flag:键]` | 旗标当前值（可当文字显示） | 随进度变的称谓、计数 |
| `[tag:item:物品id]` | 物品显示名 | 「你摸了摸 [tag:item:铜钱]」 |
| `[tag:npc:角色id]` | 角色名；可用 `[tag:npc:@context]` 指当前对话 NPC | 「[tag:npc:关二狗] 哼了一声」 |
| `[tag:player]` | 玩家显示名 | 「[tag:player]，回来啦？」 |
| `[tag:quest:任务id]` | 任务标题 | 「关于 [tag:quest:寻狗记] 的线索」 |
| `[tag:rule:规矩id]` | 规矩条文摘要 | 「违反 [tag:rule:rule_no_answer_name] 者…」 |
| `[tag:scene:场景id]` | 场景显示名 | 「前往 [tag:scene:茶馆]」 |

**怎么插：** 富文本框边 **插入引用** 按钮 → 选类型 → 选条目 → 自动生成正确写法，比手打不易错。

**手打注意：** 标识必须与游戏里登记一致；错了可能显示空白，编辑器不一定当场报错。

## 插图标签 `[img:路径或短名]`

| 项 | 说明 |
|---|---|
| **干什么** | 正文内嵌一张图片 |
| **哪里能点按钮插** | **仅 [档案](../editors/panels/archive)** 正文 |
| **别处** | 手打标签**会随正文保存**，编辑器不校验、不引导；运行时是否渲染取决于该展示点是否支持插图——正式内容优先只在档案里用按钮插入 |

雾津见闻录配图、书籍插图：在档案里用按钮插。

---

## 哪些字段不能插标签

以下多为**纯标识**或内部引用，界面是单行文本，**不可**插引用标签：

- 各种内部 id 字段（任务 id、场景 id、物品 id 等）
- 规矩碎片上只读的规矩引用行

对白、描述、任务说明、规矩正文、文本库字符串值——走富文本框的**可以**插。

## 快速示例

**对白：**

> [tag:npc:关二狗]：这 [tag:item:桃木片] 不是你的吧，[tag:player]？

**档案（带图）：**

> 雾津渡口旧照如下：[img:images/backgrounds/day_alley_bg.png]

操作：图对话里点 **插入 → 物品 / NPC / 玩家**；档案里另可 **插入 → 插图**。

## 常见问题

**为什么标签显示成一片空白或原样的方括号？**
常见原因：少了 `tag:` 前缀；或标识与登记名对不上。用「插入引用」按钮生成的标签基本不会出这个问题。

**同一个场景里有两个同名 NPC，标签会不会引用错？**
用 `[tag:npc:@context]` 指「当前正在跟你说话的那个」；或用明确的角色 id。

**旗标标签显示的是旗标的名字还是它的值？**
`[tag:flag:…]` 显示的是这个旗标**当前的值**，不是旗标本身的名字。

**插图标签能不能用在对白里？**
技术上会保存；编辑器**只在档案条目**给插入按钮。正式内容尽量只在档案里用插图标签。

**改了一个物品/NPC 的登记名字，所有引用它的文案要不要跟着改？**
不用。标签存的是标识，运行时按当前登记名动态换。
