富文本标签速查

玩家看到的对白、描述、档案正文里，用 [类型:标识] 插入动态引用——运行时自动换成当前名字、数值或链接，改登记名全文跟着变。

入门见 怎么写带引用的文本。

八种引用标签​

标签写法引用什么典型句子
[string:…]文本库里的固定条目系统统一称呼、格式化用语
[flag:…]旗标当前值（可当文字显示）随进度变的称谓、计数
[item:…]物品显示名「你摸了摸 [item:狗哨]」
[npc:…]角色名（可加 @场景实例 消歧）「[npc:关二狗] 哼了一声」
[player:…]玩家化身名、称谓等「[player:name]，回来啦？」
[quest:…]任务标题「关于 [quest:寻狗记] 的线索」
[rule:…]规矩条文摘要「违反 [rule:城规三则] 者…」
[scene:…]场景显示名「前往 [scene:雾津码头]」

怎么插： 富文本框边 插入引用 按钮 → 选类型 → 选条目 → 自动生成，比手打不易错。

手打注意： 标识必须与游戏里登记一致；错了可能显示空白，编辑器不一定当场报错。

条件里的旗标值​

条件编辑器里，旗标比较值也可以插 [tag:…] 类引用（与上表同源），让条件跟动态文案对齐。

插图标签 [img:短名]​

项说明
干什么正文内嵌一张已登记图片
短名从哪来叠图 面板登记的短名
哪里能点按钮插仅 档案 正文
别处手打标签运行时可能显示，但编辑器不校验、不引导——属于盲区行为，保存还可能被抹（见 危险区）

雾津见闻录配图、书籍插图：在档案里用按钮插；别在场景热区手写赌保存。

哪些字段不能插标签​

以下多为纯标识或内部引用，界面是单行文本，不可插引用标签：

  • 各种内部 id 字段（任务 id、场景 id、物品 id 等）
  • 规矩碎片上只读的规矩引用行

对白、描述、任务说明、规矩正文、文本库字符串值——走富文本框的可以插。

常见出现位置​

面板字段
图对话台词、选项文案
物品描述、动态描述
档案条目正文（可插图）
任务标题、说明
规矩各层正文
遭遇选项与结果文字
文本库字符串值
场景热区显示标签

快速示例​

对白：

[npc:关二狗]：这 [item:狗哨] 不是你的吧，[player:name]？

档案（带图）：

雾津渡口旧照如下：[img:wuji_dock_sketch]

操作：图对话里点 插入 → 物品 / NPC / 玩家；档案里另可 插入 → 插图。

常见问题​

为什么标签显示成一片空白或原样的方括号？ 标识没对上——十有八九是手打的名字和登记的名字差一个字，或者引用的条目根本没登记过。用「插入引用」按钮生成的标签基本不会出这个问题，手打的记得对照登记表逐字核对。

同一个场景里有两个同名 NPC，标签会不会引用错？ 用 @场景实例 的写法消歧——比如同一张图里有两个「守卫」，就分别写清楚指向哪一个实例，不加消歧默认按名字找第一个匹配。

旗标标签显示的是旗标的名字还是它的值？ [flag:…] 显示的是这个旗标当前的值（比如好感数字、称谓文本），不是旗标本身的名字——想在文案里提旗标代表的意思，得靠旗标本身设计成什么值。

插图标签能不能用在对白里？ 技术上运行时认，但编辑器只在档案条目给你按钮插入引导；对白、任务说明等其它富文本框手打插图标签也可能显示出来，但编辑器不校验短名是否存在、也不提醒你——出问题不容易第一时间发现，正式内容尽量只在档案里用插图标签。

改了一个物品/NPC 的登记名字，所有引用它的文案要不要跟着改？ 不用。标签存的是标识，不是显示文字，运行时按当前登记的名字动态换——改一次登记名，所有引用处全部自动跟着变，这也是用标签而不是手打死文字的意义所在。
