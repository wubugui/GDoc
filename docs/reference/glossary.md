# 术语表

编排雾津内容时反复出现的词——每条一句**大白话**解释，再补一句**在哪会用到**，按**策划能直接用的说法**讲，不绑实现细节。

---

## 世界与场景

| 术语 | 大白话 | 在哪会用到 |
|---|---|---|
| **场景** | 一张可走动地图：背景范围、BGM、滤镜、热区、区域、NPC、出生点。玩家在场景里操控角色探索。 | 摆一整片地方（雾津码头、城隍庙）都从新建场景开始，见 [场景面板](../editors/panels/scene)。 |
| **热区** | 地图上「可交互的一坨」——牌匾、可捡物、通往别处的门、遭遇入口等。有类型：观察、拾取、转场、遭遇等。 | 场景面板里画一个点或一小块范围，让玩家能点、能捡、能走进去。 |
| **区域** | 脚踩进去才触发的地面范围（多边形）。可挂进入/停留/离开动作。分标准区与深度地板区。 | 玩家走到城隍庙门口自动触发一段台词，就是靠区域的进入动作。 |
| **出生点** | 玩家进入本场景时落下的坐标；转场可指定落到哪个出生点。默认出生点不可删。 | 场景面板里设，转场型热区和切换场景动作都要指定落到哪个出生点。 |
| **转场** | 从当前场景切到另一张场景——常由转场型热区或动作触发。 | 关二狗推门进屋、坐船离开雾津码头，都是转场。 |
| **位面** | 同一物理场景上的「重叠世界层」——如现世/里世界。控制谁能看见、能否捡物对话、镜头与扣血等。 | 夜探雾津切到「里世界」这类玩法，靠 [位面面板](../editors/panels/plane) 和叙事状态机联动。 |
| **深度** | 画面远近关系：离镜头近的挡远的。深度地板区让玩家能走到物件「后面」。 | 想让角色能走到摊子后面、被招牌挡住，要靠 [场景深度](../editors/render-domain/scene-depth-editor)。 |
| **视差** | 多层背景按不同速度移动，营造镜头扫街的纵深感。 | 过场里镜头扫过雾津街景时，用 [视差编辑器](../editors/render-domain/parallax-editor) 做出的层次感。 |
| **滤镜** | 整张图的色彩氛围变换（偏暖、压暗等），场景可指定进图时用哪个。 | 场景顶层字段里选滤镜 id，配合 [滤镜工具](../editors/render-domain/filter-tool) 调色。 |
| **碰撞范围 / 碰撞多边形** | 热区、NPC 身上一圈看不见的边界，划定玩家在哪能真正碰到它、被它挡住。在画布上直接拖点画出来。 | 场景面板里画热区、NPC 的边界时会用到，决定玩家撞不撞得上。 |
| **交互范围** | 玩家要离得多近，热区/NPC 才会响应互动——太远点了也没用。 | 场景面板热区/NPC 的检视器字段，调「离多近才能点」。 |
| **自动触发** | 不用玩家主动点，靠近或满足条件就自动发生，省去「请按 E」这一步。 | 场景热区上的开关，配合区域进入更常见，但热区也能用。 |

## 角色与表现

| 术语 | 大白话 | 在哪会用到 |
|---|---|---|
| **NPC** | 场景里的角色实例：站哪、朝哪、什么动画、聊哪张图对话、巡不巡逻。身份在角色登记里定义，场景里放实例。 | 关二狗、李天狗这些能在场景里走动、说话的角色，都是 NPC 实例。 |
| **玩家化身** | 玩家操控角色的外观动画包与立绘等——全局配置管大方向，专门面板管细节。 | 换装、变装剧情用「设玩家化身」动作，细节在 [玩家化身面板](../editors/panels/avatar) 配置。 |
| **巡逻** | NPC 沿折线路径来回走；可当场停或写入存档永久停/启。 | 场景里给 NPC 拖一条巡逻折线，让他自己来回走动。 |
| **叠图** | 屏幕上叠加的登记图片（插图、线索特写），短名在叠图面板登记。 | 过场特写、文档揭示、档案插图都要先在 [叠图面板](../editors/panels/overlay) 登记短名才能引用。 |
| **过场** | 不用玩家乱走的时间轴演出：黑场、字幕、运镜、对白、插图等，排好后用动作调用。 | 章节开场、大段剧情演出走 [过场面板](../editors/panels/cutscene)，排好后用「开始过场」动作触发。 |
| **信号 Cue** | 打包好的一串表现动作，一处触发统一播。 | 一段固定的「惊讶+音效+抖动」组合反复用到好几处时，打包成一个 Cue 省得每次重排。 |

## 叙事与对话

| 术语 | 大白话 | 在哪会用到 |
|---|---|---|
| **图对话** | 节点图式对白：台词、选项、分支、执行动作、按叙事状态分叉、结束。玩家可点选项。 | NPC 交互、热区观察转对白都挂一张图对话，见 [图对话面板](../editors/panels/dialogue-graph)。 |
| **剧本** | 一条可推进的叙事线，分多个阶段，可暴露旗标、关联图对话（关联列表只读展示）。 | 多线叙事（纸人线、寻狗线）各自是一条剧本，见 [剧本面板](../editors/panels/scenarios)。 |
| **剧本行** | 剧本里更细的节拍行——条件可查「这一行触发了没有」。 | 想判断「某个评书段听完了没」这类细粒度节拍时用。 |
| **叙事状态** | 叙事状态机里的节点：章节大开关、位面点名、进出动作。正规推进用发信号，而非硬跳。 | 章节切换、位面激活的正规入口在 [叙事状态机](../editors/narrative-domain/narrative-editor-web)。 |
| **遭遇** | 选项式事件：多个选项，各有条件、消耗、结果动作与结果文字。 | 盘问、议价、危机抉择这类多选一事件走 [遭遇面板](../editors/panels/encounter)。 |
| **档案** | 游戏内书本系统：人物簿、见闻录、文档、书籍；可解锁、首次阅读动作、插图正文。 | 玩家翻的人物簿、见闻录、杂书匣都在 [档案面板](../editors/panels/archive) 编排。 |
| **文档揭示** | 模糊图随条件变清晰，并写揭示后旗标。 | 调查玩法里凑齐线索让一张模糊图变清楚，走 [文档揭示面板](../editors/panels/doc-reveal)。 |

## 规则、物品与经济

| 术语 | 大白话 | 在哪会用到 |
|---|---|---|
| **规矩** | 玩家可收集、验证的条文；分未齐、锁定提示、已验证等层；可有碎片线索。 | 「城规三则」这类玩家要慢慢拼全、验证的条文，走 [规矩面板](../editors/panels/rule)。 |
| **旗标** | 布尔/数值/文本型进度开关——条件常查它，动作常改它。分静态与按规则生成的模式旗标。 | 几乎所有条件判断和进度记忆都靠旗标，见 [旗标面板](../editors/panels/flags)。 |
| **任务** | 任务日志条目：前置、完成条件、接取/完成动作、下一环连线。 | 寻狗记这类任务线，从接取到交差都在 [任务面板](../editors/panels/quest) 编排。 |
| **物品** | 背包可持有物：名称、描述、堆叠、动态描述（按条件换文案）。 | 狗哨、香烛这类能拿到手的东西，登记在 [物品面板](../editors/panels/item)。 |
| **商店** | 价目表：哪家卖什么、多少钱；用动作可打开商店 UI。 | 香烛铺、当铺这类能买卖的地方，走 [商店面板](../editors/panels/shop)。 |
| **地图** | 世界地图上的场景节点与连线，可设解锁条件、拖坐标。 | 玩家翻开的世界地图、场景间的解锁关系，走 [地图面板](../editors/panels/map)。 |

## 动作、条件与文本

| 术语 | 大白话 | 在哪会用到 |
|---|---|---|
| **动作** | 「这一刻游戏要发生的事」——给物、切场景、播对白、改旗标等；列表挂在任务、热区、对白节点等处。 | 几乎每块面板都有动作列表，详见 [动作大全](./actions-catalog)。 |
| **条件** | 「当前状态是否满足」——旗标、任务、剧本、叙事状态等叶子，加全部/任一/取反组合。 | 任务前置、选项是否可点、地图是否解锁，详见 [条件类型速查](./conditions-catalog)。 |
| **富文本 / 引用标签** | 正文里 `[类型:标识]`，运行时换成动态名字或值。 | 对白、描述、档案正文里插名字、物品、图片，详见 [富文本标签速查](./text-tags)。 |
| **前置条件** | 尚未满足则内容不可见或不可接。 | 任务、地图转场、档案条目等「要不要显示/能不能进」的判断。 |
| **完成条件** | 满足则任务算交差。 | 任务面板里判断「这条任务算不算完成」。 |

## 小游戏

| 术语 | 大白话 | 在哪会用到 |
|---|---|---|
| **小游戏** | 独立玩法片段（水域捞物、糖画转盘、扎纸等），由动作或热区拉起，自有规则与结算动作。 | 河边摸鱼、街头糖画转盘、丧葬扎纸这类穿插玩法，各自有对应面板。 |
| **临场长按** | 蓄力条 QTE：按住填满、松手或中断各有动作。 | 推门、稳烛火这类需要玩家按住蓄力的场景，走 [临场长按面板](../editors/panels/pressure-hold)。 |

## 编辑器相关

| 术语 | 大白话 | 在哪会用到 |
|---|---|---|
| **面板** | 主编辑器左侧一项对应一块编辑界面（场景、任务、图对话等）。 | 打开主编辑器后，左侧导航树点开的每一项都是一块面板。 |
| **重建区** | 保存时整段重写，面板没填的额外内容会丢。 | 判断「这里能不能多写点东西」时先查 [危险区](./danger-zone)。 |
| **盲区** | 游戏认得，主编辑器没有控件维护——应换专项工具或走升级。 | 场景深度、旗标迁移块这类字段落在这里，详见 [危险区](./danger-zone)。 |
| **Apply / 保存** | 把当前面板改动写入工程；`Ctrl+S` 存当前，`Ctrl+Shift+S` 存全部。 | 每次改完内容都要做这一步，否则改动不会写进工程，见 [快捷键](./shortcuts)。 |
| **运行预览** | 主编辑器内 `F5` 跑游戏验证改动。 | 改完内容想立刻看效果时按一下，见 [运行预览](../editors/main-editor/run-preview)。 |
| **动作总表** | 只读动作类型目录，不能在这里编排实例。 | 想核对某个动作官方叫什么名字时去查一眼，实际编排还是回各业务面板。 |
| **登记册** | 有些内容（比如小游戏各关卡、角色）先在一张总表里报个名字和文件位置，再到各自的详情面板里填内容——总表管「有哪些」，详情面板管「长什么样」。总表里删了一行，实际的详情文件不会跟着自动清掉，得手动收拾。 | 小游戏实例、角色登记这类「先报到、再填详情」的面板都是这个结构。 |
