# 术语表

编排雾津内容时反复出现的词——按**策划能直接用的说法**解释，不绑实现细节。

---

## 世界与场景

| 术语 | 大白话 |
|---|---|
| **场景** | 一张可走动地图：背景范围、BGM、滤镜、热区、区域、NPC、出生点。玩家在场景里操控角色探索。 |
| **热区** | 地图上「可交互的一坨」——牌匾、可捡物、通往别处的门、遭遇入口等。有类型：观察、拾取、转场、遭遇等。 |
| **区域** | 脚踩进去才触发的地面范围（多边形）。可挂进入/停留/离开动作。分标准区与深度地板区。 |
| **出生点** | 玩家进入本场景时落下的坐标；转场可指定落到哪个出生点。默认出生点不可删。 |
| **转场** | 从当前场景切到另一张场景——常由转场型热区或动作触发。 |
| **位面** | 同一物理场景上的「重叠世界层」——如现世/里世界。控制谁能看见、能否捡物对话、镜头与扣血等。 |
| **深度** | 画面远近关系：离镜头近的挡远的。深度地板区让玩家能走到物件「后面」。 |
| **视差** | 多层背景按不同速度移动，营造镜头扫街的纵深感。 |
| **滤镜** | 整张图的色彩氛围变换（偏暖、压暗等），场景可指定进图时用哪个。 |

---

## 角色与表现

| 术语 | 大白话 |
|---|---|
| **NPC** | 场景里的角色实例：站哪、朝哪、什么动画、聊哪张图对话、巡不巡逻。身份在角色登记里定义，场景里放实例。 |
| **玩家化身** | 玩家操控角色的外观动画包与立绘等——全局配置管大方向，专门面板管细节。 |
| **巡逻** | NPC 沿折线路径来回走；可当场停或写入存档永久停/启。 |
| **叠图** | 屏幕上叠加的登记图片（插图、线索特写），短名在叠图面板登记。 |
| **过场** | 不用玩家乱走的时间轴演出：黑场、字幕、运镜、对白、插图等，排好后用动作调用。 |
| **信号 Cue** | 打包好的一串表现动作，一处触发统一播。 |

---

## 叙事与对话

| 术语 | 大白话 |
|---|---|
| **图对话** | 节点图式对白：台词、选项、分支、执行动作、按叙事状态分叉、结束。玩家可点选项。 |
| **剧本** | 一条可推进的叙事线，分多个阶段，可暴露旗标、关联图对话（关联列表只读展示）。 |
| **剧本行** | 剧本里更细的节拍行——条件可查「这一行触发了没有」。 |
| **叙事状态** | 叙事状态机里的节点：章节大开关、位面点名、进出动作。正规推进用发信号，而非硬跳。 |
| **遭遇** | 选项式事件：多个选项，各有条件、消耗、结果动作与结果文字。 |
| **档案** | 游戏内书本系统：人物簿、见闻录、文档、书籍；可解锁、首次阅读动作、插图正文。 |
| **文档揭示** | 模糊图随条件变清晰，并写揭示后旗标。 |

---

## 规则、物品与经济

| 术语 | 大白话 |
|---|---|
| **规矩** | 玩家可收集、验证的条文；分未齐、锁定提示、已验证等层；可有碎片线索。 |
| **旗标** | 布尔/数值/文本型进度开关——条件常查它，动作常改它。分静态与按规则生成的模式旗标。 |
| **任务** | 任务日志条目：前置、完成条件、接取/完成动作、下一环连线。 |
| **物品** | 背包可持有物：名称、描述、堆叠、动态描述（按条件换文案）。 |
| **商店** | 价目表：哪家卖什么、多少钱；用动作可打开商店 UI。 |
| **地图** | 世界地图上的场景节点与连线，可设解锁条件、拖坐标。 |

---

## 动作、条件与文本

| 术语 | 大白话 |
|---|---|
| **动作** | 「这一刻游戏要发生的事」——给物、切场景、播对白、改旗标等；列表挂在任务、热区、对白节点等处。 |
| **条件** | 「当前状态是否满足」——旗标、任务、剧本、叙事状态等叶子，加全部/任一/取反组合。 |
| **富文本 / 引用标签** | 正文里 `[类型:标识]`，运行时换成动态名字或值。见 [富文本标签速查](./text-tags)。 |
| **前置条件** | 尚未满足则内容不可见或不可接。 |
| **完成条件** | 满足则任务算交差。 |

---

## 小游戏

| 术语 | 大白话 |
|---|---|
| **小游戏** | 独立玩法片段（水域捞物、糖画转盘、扎纸等），由动作或热区拉起，自有规则与结算动作。 |
| **临场长按** | 蓄力条 QTE：按住填满、松手或中断各有动作。 |

---

## 编辑器相关

| 术语 | 大白话 |
|---|---|
| **面板** | 主编辑器左侧一项对应一块编辑界面（场景、任务、图对话等）。 |
| **重建区** | 保存时整段重写，面板没填的额外内容会丢。 |
| **盲区** | 游戏认得，主编辑器没有控件维护——应换专项工具或走升级。 |
| **Apply / 保存** | 把当前面板改动写入工程；`Ctrl+S` 存当前，`Ctrl+Shift+S` 存全部。 |
| **运行预览** | 主编辑器内 `F5` 跑游戏验证改动。 |
| **动作总表** | 只读动作类型目录，不能在这里编排实例。 |
