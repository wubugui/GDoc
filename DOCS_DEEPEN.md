# 文档加深指导书（写手必读）

> 目标：把每一页从「够用」做到「够深」。**先入门后进阶**：新手照着能上手，老手往下能挖到底。
> 本指导书配合 [DOCS_PLAN.md](./DOCS_PLAN.md) 第 0 节「写作铁律」使用——铁律不变，这里只讲**怎么加深**。

## 一、铁律回顾（再违反一次就作废）

- **零代码**：不准出现源文件名、模块名、类名、字段内部结构、代码块（launch 命令除外）、代码链接。只写「怎么用」。
- 第二人称、大白话、雾津真实例子、术语给解释。
- 每页配示意图（Mermaid 或既有 SVG）。

## 二、加深模板（每页都改成这个骨架）

保留原页有用的内容，**重排成下面的结构，并把每节写厚**：

```
（若已有题图 PageBanner，保留）

# 标题

一两句话：这页讲什么、读完你能做到什么。

## 这是什么（30 秒看懂）
- 大白话说清它管什么、解决什么问题；配一个雾津场景的类比。

## 入门：手把手做第一次
- 最常见的那一件事，从头到尾编号步骤（点哪里、填什么、怎么保存、怎么验证）。
- 一张流程 Mermaid 示意图。
- 一个「照着抄」的雾津小例子（用真实角色/场景：关二狗、李天狗、雾津街头、城隍庙、糖画摊…）。

## 进阶：每一项都讲透
- 把这块/工具**所有能力、所有可填项**逐条讲清：**是什么用途、能填什么范围、填了会怎样**（用户视角，不碰内部结构）。
- 和别的面板/工具怎么配合（互相引用、先做谁后做谁）。
- 批量做法、效率窍门、老手才知道的技巧。

## 危险区与边界
- 这里改了会丢什么、哪些字段编辑器根本够不到、什么操作不可逆。
- 引到 [危险区](/docs/reference/danger-zone) 和相关参考。

## 常见问题
- 3–6 条真实会遇到的 Q&A（「为什么我改了没生效」「东西不见了」这种）。

## 相关
- 链到上下游页面、对应教程、参考。
```

**深度目标**（正文字数只是参考，重点是「有没有把每一项讲透」）：
- 主编辑器面板页：**4000–6000 字**，进阶节要覆盖该面板每一个可填项。
- 专项工具页 / 工作台 Tab：**3000–4500 字**。
- 教程页：**2500–4000 字**，步骤要细到不会卡。
- 概念页（动作/条件/富文本）：**4000+ 字**，把每类用法讲全。
- 参考页（各种目录）：**尽量全**——动作大全就该把每个动作列全并逐个说用途。
- 玩家手册：**2000–3500 字**，玩家视角，别剧透主线。

## 四、纪律

- **保留并延续既有风格**（雾津调性、题图、Mermaid），只做加厚，别推倒重写得面目全非。
- 不确定某个功能到底怎么用、读了素材还是拿不准 → **停下来回来问派活的人（我）**，别瞎编。宁可标「（待核实）」也不要写错。
- 只改分配给你的页；不动别人的页、不动配置。
- 写完自查：① 有没有代码符号漏进去 ② 入门/进阶两层齐不齐 ③ 有没有雾津实例 ④ 有没有示意图。
